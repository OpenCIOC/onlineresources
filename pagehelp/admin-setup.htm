The Database Setup page is available only to Super Users. It serves as the main menu for various setup and help related options in all modules of the software. Depending upon your level of access (i.e. whether you are a Super User for one module or more than one), some of the options on this page may be unavailable to you.

General Setup Options

Area for modifying setup options that apply to the entire database, including: setting admin area templates, enabling or customizing some software features such as Offline Tools, setting default (public) Views, etc.

Agencies

Allows you to configure record owner information for agencies that manage records or have users in the database.

Manage Users

The Manage Users area allows you to search for, update, and create new Users.

Design Templates

Design Templates control the style / look of the database, including colours, logos, style sheets, headers and footers, etc.

Template Layouts

Template Layouts are used within a Design Template to provide custom formatting for headers, footers, and main menu / search pages.

Record Inclusion Policies

Allows you to edit the Inclusion Policies that are provided to someone suggesting a new record for the database.

Social Media Types

Social Media Types setup allows you to modify the list of Social Media Types, including deciding which types will appear on update and feedback forms. Note that because Social Media icons are required and must be installed in the appropriate directories within the software, you should consult with the software developers about adding new icons if you wish to add an additional Social Media Type.

Domain Name Mapping

Allows you to review the list of domain names associated with your database, and map them to specific Views. Note that domain name mappings apply to the public, and not logged-in users. Logged-in users are free to access the database using any domain name, since the Views they access are determined by their account security settings and not the domain name they use.

Mapping Systems

Mapping Systems setup allows you to integrate various mapping services (e.g. Google Maps, MapQuest, etc) with the software.

Mapping Categories

Mapping Categories allow you to specify custom labels for each colour of map pin when using the built-in Google Maps tools (geocoding).

Views

Views control what a user can see depending upon their login (or if they have no login - the "public"). Views control which records are available, which fields can be viewed or edited, whether the alert column is visible, what footer and side menu messages appear, etc.

User Types

User Types define what functions an individual can/cannot perform in the database such as adding records, updating records, whether alerts are seen around feedback/comments, whether user is Super User, and other function related options.

Field Display

Allows you to edit the field labels and the order these fields appear in for records in the database.

Hidden Fields

Allows you to hide fields that you do not use. Hidden fields will not be given as options for new field selection on setup forms such as Views, Print Profiles, Export Profiles, etc. but hiding a field will not remove it from a View where it has already been added.

Field Display Yes/No Values

For radio button fields, specify the values to be displayed in the record when True (Yes) and False (No) are selected.

Extra Field Setup

Extra Field Setup allows you to create new extra fields of various types and, where applicable, define the number of characters that can be entered into text fields (max. length). Note: in the Volunteer software, only the length of 2 pre-defined Extra fields may be modified.

Excel Output Profiles

Excel Output Profiles allow you to create a custom output that can be opened by Microsoft Excel; select which fields are output, what order the records occur in, and which View(s) may access the profile.

Export Profiles

Export Profiles allow you to configure different profiles for exporting records for use by external systems, including other CIOC databases using the CIOC Sharing Format (Community Information module only). Export Profiles allow you to define which fields, Distribution and Publication Codes that you want to share.

Print Profiles

Print Profiles allow you to create custom templates for printing lists of records.

Privacy Profiles

Privacy Profiles allow you to configure different profiles for concealing private data in individual records from the public or unauthorized users, even if they would normally see that field in their View. It is extremely important to use Privacy Profiles sparingly and carefully, especially with records that are shared with external partners. Do not use privacy profiles unless you fully understand the challenges involved.

Email Update Text

Allows you to create default values for the customizable sections of an Email Update Request (Community Information or Volunteer modules) or and Email Update All Opportunities Request (Volunteer module only).

Age Groups for Searching

Create age group ranges for use on the basic search and child care search forms (Community Information module only).

Activity Statuses

Customize the pre-defined status values for use in the ACTIVITY_INFO field.

Sharing Profiles

In a mult-member database, Sharing Profiles allow you to create one or more contracts for sharing record data with other CIOC members in your database.

Offline Tools

If Offline Tools services are enabled in the database, this section allows management of which User Types have access to registered computers / servers that have installed the CIOC Offline Tools.

Vacancy Setup

Customize various pre-defined values and form setup items for use in the VACANCY_INFO field.

Checklist Management

It is important to carefully read the information contained on the various Edit Checklist / Drop-Down Values pages. With two exceptions (Street Type and Postal Box Type), changes made to checklists and drop-down values on these pages impact not only on the available values in the drop-down menus and checklists / radio buttons, but will cause changes in all records in the database that use those values.

Click on the checklist / drop-down type of your choice on this page to move to an "edit" page where you can edit, delete and add new values for various drop-down menus and checklists that appear in the database.

Classification System Management

Classification systems are used for categorizing and searching records. Many types of classification systems are used across different databases and may come from external (official) sources. Note that some types of classification systems are also available from the main menu (Publications, Taxonomy) because all or part of the management tools are available to users who are not Super Users. These systems are also linked here for convenience and consistency.

Page Titles

Allows you to override the default titles of a some of the pages in the software. Override is currently available for only a small number of pages.

Page Messages

A Page Message allows you to insert some information on a page, such as special instructions or announcements. This method is useful when the information needs to occur on some pages but not others; information that should appear on all pages should be adjusted in the Design Template. You may also select the View(s) in which you want to display the messages.

Field Help

Allows you to edit the field help that appear in record update and feedback forms. These are the links that appear beside data entry fields that provide information about data entry rules / guidelines for that field.

Search Tips

Allows you to edit the text for the Search Tips help page that is linked from the Main Menu / Basic Search page.
