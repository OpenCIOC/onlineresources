Activation Recommendation Report

Unlike the Multi-level Activation Report, this tool can assist you in making activation changes directly. When the tool is loaded, you will be presented with a form showing the options available for the tool and the results of the tool as run with the current (default) options.

Also unlike the Multi-level Activation Report, this report only makes recommendations on the obvious changes that can be made without the editorial support of a trained data manager. That means it does not include items such as a branch where records are indexed at multiple levels, since a decision must first be made by an appropriate editor about which level will be preferred for future indexing.

What types of recommendations does the tool make?

  • The tool will recommend deactivation when there is an Active Term with no records indexed to that Term, and there is another Active Term on the same branch that does have records indexed to it.
  • The tool will recommend activation when there is an Inactive Term with records indexed to it (this will be a non-optional change if you submit the form).
  • The tool will recommend activation when you have an Inactive Term whose Parent Term and one or more Child Terms are Active (this will also be a non-optional change if you submit the form). Although it is not advised to activate at multiple levels, if you choose to have multi-level activations you must do so in a contiguous block.
  • If the Recommend Activations option is chosen, the tool will also recommend activating Terms from branches on which no other Term is active. If applicable, the tool will recommend using the level on which other sibling Terms are already activated.

Setting Report Options

Before submitting your desired changes, you may adjust the recommendation options and regenerate the report.

  • Show global activations - this option applies to a multi-member database, to distinguish between Global Activations that apply to all members in the database, and Local Activations that apply only to your membership.
  • Inactivate unused Terms with used terms on the same branch - if this option is not selected, no deactivations will be recommended.
  • Include Shared Records - in a multi-member database, use this option to make recommendations take into account records that have been shared with you by other members.
  • Rollup Low Level Terms - when a term recommended for deactivation has an Active Parent that is being used, roll up the Term as a use reference for the Parent Term rather than deactivate it.
  • Exclude Y Branch - because multi-level activations are permitted in the Y (Target Populations) branch, this option ignores deactivation recommendations for this branch.
  • Recommend Activations - this option recommends new activations for branches on which there are no Active Terms. It is particularly useful after a Taxonomy version update, to highlight new Terms.

Reviewing the Report

The report is divided into two sections:

  • the first section lists deactivation recommendations and mandatory activations (if any)
  • the second section lists activation recommendations

Next to the Code and Term name, the action describes whether the Term's activation status will set to Active, Inactive, or Roll-up.

Submitting Changes

You may choose to accept some or all of the recommendations provided. By default, all recommendations are checked - but you can check or uncheck items to select only the ones you want. You may use the "Uncheck All" button to start from a blank selection. Recommendations without a checkbox break basic software rules (such as having an Inactive Term that is indexed to records) and are therefore mandatory. These mandatory changes will be applied automatically if you submit the form. Once you submit the form by clicking "Perform All Selected and Mandatory Changes", your activation changes will be applied immediately.
