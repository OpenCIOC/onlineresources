The main Import Record Data page is the main menu for the Import Tool. The Import Tool allows you to upload XML files in the CIOC Sharing Format that have been exported from other databases. The menu on this page gives you list of existing export files to work with and provides a link to Load New Dataset to upload a new export file.

Important Note: The Update / Add Records functions of the Import Tool require a great deal of resources from the server, and can interfere with the normal operation of the database. It is strongly advised that the tool not be used during peak operating times (9am EST - 5pm EST). Use of the Import Queue utility may assist you in planning your Imports for non-peak times. The process of loading the file does not cause a performance problem on the server, and can be done at any time; however, it is recommended that you not leave a large number of unprocessed export files loaded into the database, due to space considerations.

Loading a New Dataset

To work with a new export file, click the Load New Dataset link; a new pop-up window will open with the file-loading tool. You must use a current, valid CIOC Sharing Format XML file; the tool does not accept any other format of data, and may not accept a file from an older or newer version of the software.

From within the pop-up window, click the "Browse" button to find the export file on your computer that you want to load. Using the "Import As" field, you can optionally give the file you are loading a friendly name to help you remember where it came from and when, i.e. "FindHelp Bilingual Data, Jan 2007". If you do not specify a name, the regular filename will be used when displaying information about the file. It is possible to load the same file twice, and to give two files the same "friendly" name, so be thoughtful when loading and naming files.

After selecting the file on your computer, select "Upload File". The file will be validated to ensure it is a valid CIOC Sharing Format XML file. If there are any issues validating the file, you will receive an error report, and some or all of the file will not be loaded. Report any errors to the person who sent the export file and ask them to make a new export. If you receive errors but some records appear to have loaded successfully, you may import those records as they have passed the validation step.

Working with an Existing Dataset

Once a dataset has been loaded, it will appear in a list of datasets on the main menu of the Import Tool. Records to be imported will be grouped under Add or Update, and records that have been imported will be under Completed; record counts will appear next to each link. To add or update records, click on the appropriate link to be taken to the Add / Update page, or click "Queue" to add the import file to the Import Queue to be imported at a later time. Files in the Import Queue can be processed by clicking the "Process Now" link; the Queue will process an unlimited number of files, both additions and updates, for up to 12 hours at a time. To view a report about records that have already been imported, click on the Completed link.

A report about the file you loaded is available under the "More Info" link. It is recommended that you carefully review the More Info report before you run any import. The report will indicate the filename, friendly name, number of records to add or update, which language the records are available in, the Publication and/or Distribution Codes included, and the fields that will be updated. It is important to make sure the file includes the data you expect, so you do not accidentally overwrite fields or codes where you wish to preserve your own data. You will have the opportunity to view data for individual records during the Add / Update process.

You can only Add or Update a record from a loaded file once, as the data you loaded is replaced with an import report after the record has been imported; if you need to redo an import, you must reload the export file.

Once you have completed importing records, or if you do not require a dataset any more, you can delete the dataset using the "Delete Dataset" link. Note that the import report will no longer be available once you delete the dataset, so if it contains any information that needs to be kept for documentation purposes you should save or print the import report available through the Completed link.

You may see some records marked as "Cannot Import"; this occurs if you are trying to import a record with the same record number as a record that is owned by another CIOC Member in your database. Rather than importing the record, it must be shared with you by the member using a Sharing Profile. If you wish to have your own copy of the record that is owned by you, you must manually change the record number by opening the XML file in a text editor and supplying a new unique record number before you load the file.

There are a number of resources available on the CIOC Community Site to help you with sharing your records. Access the Resources area, or post on the Message Board, for more information.
