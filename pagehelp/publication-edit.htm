The "Add / Edit Publication" page allows you to either edit an existing Publication or add a new Publication to the database. If you are editing an existing Publication, the existing settings will be in the form. If you are adding a new Publication, you will be presented with a blank form. The only required setting for any Publication is the unique code that will represent the Publication.

Publication Data Elements

Status

If you have chosen to edit an existing Publication, this section will display information about that Publication. If you are adding a new Publication, this section will not appear. Inside the Status section is information about whether the Publication is currently in use, either by a database View or in Organization / Program records. The record count given is for the number of records associated with this Publication in the entire database (not just the current View).

If the Publication is being used in records, you will see a "Search" link which will allow you to bring up a results set which shows you the records in your current View that belong to this Publication. If the Publication is being used, either by a database View or in individual records, you cannot delete it. If it is not being used, you will have the option of deleting the Publication using the "Delete" button that will appear at the bottom of the form.

Created/Modified Information

Created and Modified dates / names are set automatically by the software as records are added and updated.

Code

The Publication Code is a unique, descriptive "short form" to identify the Publication. Only alpha-numeric characters and the dash "-" are allowed in this field, and the maximum length of the code is 20 characters.

Publication Notes

It is recommended that a description of the purpose of each publication code is entered into this fields as a reminder of how it is intended to be use. This will help prevent confusion or mistaken use, and make it easier to tidy up unused codes in the future.

Name

Names are sometimes used in addition or instead of codes Publication drop-down such as the"Quick List". Whether names or codes are used (or both) depends on the page being viewed, and also the settings for Publication "friendly names" from within the View setup.

Note: It is important to make this a very brief but clear description as it will impact on the width of the drop down menu that appears for the "Quick List".

Non-Public

Checking this box will prevent the Publication from being visible in Publication drop-down lists in any Views of the database that are not permitted to see non-public records. It will not, however, hide the Publication code from showing up in the Publication field of any record that has been assigned to the Publication.

Field Use

There are two types of Publication fields; the Description (a block of text up to 8000 characters) and General Headings (a custom classification system for records belonging to this Publication). General Headings can be further subdivided into a number of different variations. The full list of Publication field types are:

  • *_DESC: Publication Description (display, feedback)
  • *_HEADINGS: Public General Headings assigned to this record (display, feedback)
  • *_HEADINGS_NP: All General Headings assigned to this record (display, feedback, update)
  • *_HEADINGGROUPS: Heading Groups associated with the public General Headings assigned to this record (display)
  • *_HEADINGGROUPS_NP: Heading Groups associated with any of the General Headings assigned to this record (display)

By selecting the Allow on Display and Forms checkboxes, you will make these fields available for use on both the "Record Details" page, the "Suggest an Update" page (aka the Feedback Form), and in the case of HEADINGS_NP fields also the regular Update Form. If these checkboxes are not selected, the only place to view or edit Publication fields will be through the "Update Publications" page for each record (which is not available to the public).

Important Note: selecting a Publication for display does not automatically add it to any pages or forms; A Super User must go to Setup > Field Display and name the new fields, and then into the View setup to add it to the View(s) they want to display the field(s). However, deselecting this box will remove the field(s) from all pages and forms currently using them.

General Headings

Introduction to General Headings

What are General Headings?

General Headings allow you to classify records within a Publication. In Publication-based Views, Headings may be used to create the "Quick List" search on the basic search form. Headings are also searchable from the Advanced Search form in any View that has access to that Publication. Records must be first assigned to a Publication before they can be indexed to that Publication's General Headings.

It is possible to create custom display fields using the Headings for any given Publication. Like other classifications such as the Thesaurus or Taxonomy, these Heading display fields will be linked to allow users to perform searches for other records that use the given Heading Terms. It is not necessary for the View to be based on the Publication in order to display the Headings field for that Publication.

When using CIOC's print tools, Headings can be used to generate a topical (subject-based) record index. Some features of General Headings are solely for use in the subject index of Print Tools, including the ability to create an "Unused Term" (that points to one or more preferred Headings to be used instead), or a list of Related Terms ("See Also ...").

Taxonomy Headings vs. Regular Headings

"Regular" or traditional General Headings are manually assigned to records. It is up to the data manager to ensure that the indexing of Headings to records is maintained as records are added or changed. There are tools available to assist data managers in keeping this indexing up-to-date, including the ability to synchronize Heading indexing from one Publication to another.

For those that use the AIRS / INFO LINE Taxonomy in their database, a more automated indexing option is available. Taxonomy Headings include a component describing a Taxonomy Search; any record assigned to the Publication that matches the search criteria will automatically be indexed to the Heading. Indexing is automatically updated every time a record is added or removed from the Publication or has its Taxonomy indexing changed. All records in the Publication are re-evaluated for indexing to the Heading whenever the its Taxonomy search criteria is updated. Note: Taxonomy-based Headings cannot be manually assigned to records; they must match the given Taxonomy search criteria to be indexed.

Each Publication's Headings can be made up of a mix of Taxonomy and regular Headings.

General Headings Groups

Headings Groups allow you to cluster a number of different Headings under a broader category. Headings can be assigned to more than one Group. These Groups can optionally be made searchable through the Quick List setup for Publication-based Views; searches for a Heading Group will match records indexed to any of its Headings. Records cannot be directly assigned to Heading Groups. Click the "Add Group" button to add a new row to the table for a creating a new group entry. Adding Headings to groups is done using the Manage General Headings tools (see below).

Manage General Headings

The "Manage General Headings" section will take you to a page where you can add, edit or remove items from the list of General Headings for this Publication. Note: Make sure you submit any changes you have made in this form before going to the Manage General Headings page, or you will lose your changes.

Edit an Existing Heading

To view or edit the settings of an existing Heading, select the it from the list and click "View / Edit General Heading"

Create New Heading

Links are provided to create a new regular Heading or Taxonomy-based Heading. Clicking the links will take you to the appropriate blank data entry form for creating the new Heading.

Quick Add Taxonomy General Headings

Using a Taxonomy drill-down search, you may select a number of Taxonomy Terms at once and create a new set of General Headings based on the selected Terms. You will not be able to add Terms that already exist in your list of Headings (this includes regular Headings with the same name as the Taxonomy Term). This tool creates one Heading per Term; it does not support linked Terms or creating Headings based on multiple Terms, but you may edit the Terms list later if you require a more complex search.

Copy General Headings

If there is another Publication in the database containing all or many of the Headings you wish to use for this Publication, you can use the Copy General Headings tool to select a Publication and add some or all of the Headings into the current Publication. Although indexing Taxonomy-based Headings will begin automatically for Taxonomy-based Headings that are copied, this is not the case for Regular Headings which much be indexed manually. For regular Headings, this feature is often used effectively in combination with the record Synchronize Headings option that is part of the "Add/Remove Headings" bulk update tool available through the search results.
