Keyword Search

The main search mechanism in the software is the keyword search. Four potential keyword search fields are available, depending on what has been made available on your current search form. These include:

  • Organization Name(s) - All levels of the organization hierarchy, as well as alternate and former names
  • Subjects - All local and authorized Subjects, as well as "Used For" Terms associated with those Subjects and any Broader Terms for those Subjects.
  • Service Categories - All AIRS / InfoLine Taxonomy Terms indexed to the record, as well as Use References for those Terms and broader level terms excluding Level 1.
  • Keywords - All of the above, as well as Areas Served, Site Location, Intersection, Site Building, Contact and Executive names, Eligibility, Accessibility, Transportation, Fees, Funding, Languages, Description, Supplemental Description, NAICS, Type of Program, Type of Care, Schools in Area, and any "Extra" fields that are configured to be part of the keyword search.

General Information About Keyword Matching

The keyword search form on the main search page always matches all keywords provided (an "AND" search). If you need to do a more complex search (such as finding either "cat" or "kitten"), then use the Advanced Search form instead.

The software automatically searches many different word forms for the current language. For instance, typing in library or libraries produces the same result, so either can be used. You can explicitly prevent the searching of word forms by using quotation marks (see below).

Important note: if a record has a Privacy Profile, fields marked as private are always excluded from keyword searches, even if you have permission to access private data.

Use of Wildcard Asterisks

You can use an asterisk at the end of any search word, for example:

  • child* will find child; children; child's; childlike; childish
  • abus* will find abuse; abuser; abused; abusing
  • child* abus* will find any of the combinations outlined above

In many cases, allowing the software to use its built-in ability to search word forms is more effective than using the asterisk for simple variations of the same word (child and children, help and helping). This is because it reduces the number of unwanted extra matches, such as scan* matching scant and scandal when what was intended was just matching scan, scans, and scanning.

Use of Quotation Marks

You can search with terms enclosed in quotation marks ... thus: "child abuse" will result in any records that have both of those exact words adjacent to each other in that order and in the same field.
"child* abus*" will result in records that have "child abuse" or "child abuser" or "childish abuser" in a single field, in that order, adjacent to each other.

Note that using quotation marks turns off the software's ability to search word forms (as mentioned above). Thus, searching "library" will not match libraries.

Community Search

This function enables you to restrict the results by geography; either by where the service/program is physically located or by the geographic service area specified in the record. It is a good idea, if you are searching for a specific service in a known area, to use these limiting features to narrow your results.

Organizations with Volunteer Opportunities

This option will only be available in those communities that are using both the Volunteer and the Community Information databases and limits the results to records with Volunteer opportunities attached. This search matches only Volunteer opportunities available to you in your current View of the Volunteer software, and automatically excludes deleted opportunities.

Search by Age Group

Search by age group allows you to locate records designated as serving a particular age range. Note that the matching record only has to serve any portion of the age range: thus when searching an age range for Youth that includes ages 12-18, it would match a record that served ages 6-12 because they match on age 12. Partial years, if specified in the record or age group setup, are used for matching purposes (e.g. 3.8 years). Age groups can be configured in the setup area of the software, so your particular values may vary. Age groups may overlap with each other.

Note that if no specific ages are entered for a record, it is assumed that the record serves all ages.

Quick List Search

This search tool allows you to limit the range of the records that are searched to a pre-defined set of records, often grouped by subject and/or geographical location. Depending on how your search form is configured, you may have a drop-down (allowing one choice) or a set of checkboxes (allowing multiple selections). In most cases, your search will have to match all your selections, but it possible to set up the search form in a way that matches any one of your Quick List choices. It may not be obvious from the search form which is the case, but your search results will indicate whether all or any of your selections were searched.

More Search Options

Organizations with WWW Sites

Search results can be limited to only those records that have a website address recorded in the WWW field

Record Number

You can return one specific record in the search results according to its Record Number, that is, the three alpha, four numeric unique identifier assigned to each record: e.g. HAM2233; OAK4489; SIM3212 and so on. If you are searching by record number, do not include other search criteria because this search, by definition, can have only one record in the result; other criteria is not applicable. This is only useful if you know the record number for the organization you are searching for.

Vacancy Search

If you use the specialized vacancy fields in the software, this search will allow you to search based on vacancy or wait list availability and target populations (the type of user who can access the service).

Browsing for Records by Name or Topic

Browsing is looking through a pre-defined list of values based on an alphabetical listing.

  • Browse by Organization - allows you to review a list of services based on an alphabetical listing of the top level organization name defined in the record.
  • Browse by Subject - allows you to view an alphabetical list of Subject Terms from the Thesaurus; a count of records using each Term is included, and clicking the link will perform a search for matching records.

Links to Other Types of Searches

NAICS (North American Industry Classification) Search

If this classification system in use, a link may be provided to browse through the categories and sub-categories. NAICS is primarily used for business directories, but may be used to classify many different types of organizations.

Child Care Search

If Child Care features are enabled in your current View, the Child Care search allows searching by some or all of: keyword, community, type of care, type of program, age(s) of children, date care is required, schools in area, school escort, subsidy availability, space availability, nearby bus route(s), and geographic proximity.

Searching using Service Categories / AIRS/InfoLine Taxonomy

Some or all of the features for searching Service Categories may be available to you:

  • Browse by Service Category - browse through categories and sub-categories of the Taxonomy, including related terms. Depending on how this search has been configured, detailed descriptions of each Term may be included at some or all levels of the Taxonomy hierarchy.
  • Basic Taxonomy Search - includes more sophisticated ways for locating Taxonomy Terms, such as through a drill-down of the Taxonomy hierarchy, keyword search, search by Code, and other options.
  • Advanced Taxonomy Search - includes all search options from the Basic Taxonomy Search, as well as the ability to search terms together in various ways.
