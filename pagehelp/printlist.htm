When accessed from the main menu, the "Print Record List" page allows you to:

  • Generate a list of records formatted for printing
  • Generate a Name / Phone Index for a Publication
  • Generate a Subject Index for a Publication

When accessed from your search results, the "Print Record List" page allows you to:

  • Customize the Message for Printing Your List of Records

Printing a List of Records

Print Profiles are associated with particular Views; a Super User can create or edit Print Profiles and assign them to Views by going to Setup > Print Profiles. If you do not have any Print Profiles available in your View, you will not be able to use this function.

Step 1: Selecting your Records and Profile

If you access this page through the "Select Checkbox" action on a group of records from search results, you will have already selected the name of the Print Profile you wish to use and the records you wish to include. If you are preparing a list of records for printing from the Print Records link on the Main Menu, you should select one or more Publication codes to print a list of records that are associated with the selected code(s). Note: if you are not working from a selection of records and you do not select at least one Publication code, the software will attempt to print all the records in your View - not recommended!

After you have chosen your Profile and selected a Publication (if applicable) Click "Next" to proceed to the next step, customizing your message.

Step 2: Customizing your Message

After selecting the records you want to print and a Print Profile, you will have the opportunity to edit, add to or remove the default report message for the selected Print Profile. Depending on the Profile settings, this message might appear once at the top of the report, or before each record in the report.

Step 3: Viewing your Report

After you have customized your message, clicking "Next" will open your report in a new window. After printing or saving your report, you can close the window with the report and return to your previous window to continue working in the database.

Generating a Name / Phone Index

This feature is not available when working with a selection of records from your search results.

The second form on the print records page allows you to configure a Name Index which will provide a quick summary of the records in a particular Publication. The numbers provided in the Index correspond to the Publication Number that is automatically assigned to the record when a print list is created. Publication Numbers are distinct from the usual record number for a record; every print list that is run starts numbering its records beginning from 1; these numbers are assigned at the time the list is run, so adding, removing or renaming a record will alter the Publication Number. For that reason, it is important to co-ordinate running your print list and index so that the numbers correspond correctly. Information about how to include the Publication Number in your print report is available in the help section for Print Profiles. Besides the record number, you have the option of including several other types of information including various Phone and Email fields and Cross Reference Names.

Important Note: All Indexes are organized based on the hierarchical name structure of naming Levels 1-5. Inconsistent naming patterns may cause odd behaviour in the Index. In order to have effectively organized indexes, begin filling in names starting with Level 1 and do not leave blank entries between names (i.e., do not have an entry in ORG_LEVEL_1 and then nothing until ORG_LEVEL_3). In addition, take care in not publishing the same Cross-Reference name in multiple records; this will cause multiple entries in the Index, each associated with a different Publication Number.

Cross-Reference Names

If you include the Cross Reference Names option, any names selected to be published as a Cross-Reference will have their own listing. Cross-Reference instances always have the proper ORG_LEVEL_1 name listed in brackets beside the published Cross-Reference name. Fields available for Cross-Reference include Organization Names Level 2-5, Legal Name, and Alternate and Former Names. A Publish checkbox appears next to each name in these fields on the update form of the record.

Phone and Email Fields

You can choose to include one or more Phone or Email listings along with the basic Name Index. To select a field for inclusion, select the checkbox next to the field. You can include a lead-in Label for the field, but this is not required and you can set the Label setting to be blank if you just want to print the field's contents. If you select to apply bold formatting, the contents of the fields you selected will be printed in bold.

If you want to create a special contact list such as a Toll-Free Phone or Email Index, you can choose to only include listings for records that have entries in at least one of the fields you have selected. Note that even though only some records are included in this type of restricted Index, this will not affect the Publication Numbers; these numbers are still assigned based on the listing the Publication as a whole.

If you include at least one of these fields, a new column will be created to hold this information; one of the options on this form is the width of this column in pixels. If you find this column is too large or too small, you can adjust the width larger or smaller; however, Microsoft Word does not obey the sizing request so this option is only useful for online printing or viewing. You can manually adjust column sizes in Word to an appropriate width after opening the file.

Additional Options

Fonts

You can select from either a serif or sans-serif set of Font families, and Font sizes from 8pt-14pt. If you do not like either set of fonts given, you can save the file and change the font in Word or by opening the HTML file in an appropriate source-editing program and changing the style information at the top of the file.

Dot Leaders

You can optionally include dot leaders in your Index, which will draw a line of dots connecting a name to its Publication Number. If you are including additional fields, the dot leader will only go as far as the column containing the additional fields.

Format Output For...

In order to get reasonable results from some features of the Index, including indenting and dot leaders, you must specify whether you intend to use the resulting report for printing online or use in Microsoft Word. A report intended for Microsoft Word will look strange when viewed online, and vice versa.

Generating a Subject Index

This feature is not available when working with a selection of records from your search results.

The third form on the Print Records page allows you to generate a Subject Index for a particular Publication. The Index is based on the General Heading information associated with the Publication, so this option will only be available for Publications that have General Headings. The Index will print all Used Headings that are associated with at least one record, as well as any Unused Headings that have a reference to at least one Used Heading. More information on creating and managing General Headings is available from the help files in that section.
