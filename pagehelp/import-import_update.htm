The Add / Update Records page is used to select options for import. You have the opportunity to view the actual record data being imported prior to running the import. Once you have selected the desired settings, use the Import Record Data button to being the import process.

The Select Record Owner(s) option, available when the import file contains multiple record owners, allows you to select only specific Record Owners from the file to import. If you do not select any, data from all Record Owners in the file will be imported. Important Note: the Record Owner may be different than what is implied from the Record Number.

The In case of an ownership conflict setting allows you to manage a situation where the Record Owner of an existing record in the database does not match the Record Owner indicated in the export file. You can either keep your existing ownership information, update ownership to match the owner in the file, or not import the record(s) in question.

If the records being imported have private information (are associated with a Privacy Profile) the Privacy Profile Map allows you to associate the privacy profiles of an incoming recordset with existing privacy profiles in your database. The system will attempt to make these matches automatically. If you are unable to make a match, you must specify whether the private data should be skipped or imported anyway - when in doubt, it is best not to import private information unless you can ensure that the information will remain private. If the existing record has a different privacy profile than the incoming record, you must specify how to resolve the conflict (keep the old profile, take the new profile, or do not import the record).

The Number of Records to Import setting allows you to import records in smaller batches, which is recommended if you have a very large number of records to import, to help prevent any very long delays to others using the server.

Import Source Database Information, if selected, populates the SOURCE_DB field in the database with the value provided in the export file. This can help you to properly credit your partners who share data with you, as well as remind you where the record originated. You can select where, or if, to display the SOURCE_DB (Source Database) field in your Views by adding or removing it from the Record Details fields chosen in your View setup.
