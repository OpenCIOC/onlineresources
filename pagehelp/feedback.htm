The "Record Feedback" Page serves two purposes; to suggest a new record, or suggest changes to an existing record. The actual form and data entry requirement(s) are the same for both functions. Public (non-logged in) and logged-in users can access both of these functions. If you want the public to have access to the "Suggest a New Record" version of the Feedback Form, you will need to provide a link to a blank feedback page from the menu, header or footer of their View, since this link is not provided automatically by the software.

Unlike the form for record updates, "Feedback" does not immediately change the record. Changes made through feedback must be reviewed and accepted through the Update Form before changes will take effect.

A Super User can change which fields are available on the Feedback Form for the current View from the View setup.

Record Inclusion Policy

If you are suggesting a new record, a link to the Record Inclusion Policy will appear at the top of the form if one is available. If you give the Public a link to suggest a new record, you should ensure that you have an Inclusion Policy available. Your Record Inclusion Policy should clearly state what kinds of records are included in your database and under what conditions. A Super User can add or edit the Policy by going to Setup > Record Inclusion Policies.

Field Information

If there is help available for a field, a question mark image will appear next to the field . Clicking this image will open a pop-up window containing the field help. To customize the field help to your local needs and usage, a Super User can edit the help from the Field Help area of the Setup.

* - If a field is designated as "required", an alert-coloured asterisk will appear beside it. Note that the feedback form does not force you to fill in required data elements for a record, even if you have the "enforce required fields" option turned on in your software - the * is for informational purposes only.

Before Submitting...

At the bottom of the form are a set of required fields that must be filled out before submitting the update. This section of the form has different requirements depending on whether the person submitting feedback is logged in, what their permissions are, and the settings of the current View.

  • About You - If the user is not logged in, they will be required to enter their name and either their phone number of email address. Other information in this section is optional.
  • Source of This Information - If the user is logged in, they should specify where the information for these changes is coming from. The previous source of information will already be listed in these fields.
  • Special Instructions - Optional, if there are any general comments about the suggested changes.
  • Use of This Information - If this section is configured to be visible on the form (from View setup), this section will be required. The user must specify what authorization, if any, was given for the use of the submitted information.
  • About These Changes - All users must specify whether the suggested changes are a complete or partial update, or if the record should be removed.
  • Notifications - You will see this section if your security level allows you to suppress email notifications. Notifications will be suppressed by default (see next section for information on email notifications)
  • Security Check - If the user is not logged in, they must answer a simple question to ensure it is a real person submitting the form and not a spam robot.

Email Notifications

When feedback is submitted, an entry is created in the database for the feedback that is later used to make the suggested changes available when updating or creating the record. In addition to this database entry, a number of email notifications may be sent out (assuming they are not suppressed - see Notifications in "Before Submitting" above). The email notifications will include a summary of all requested changes to the record; each Agency should have an official policy in place about whether these emails are kept for future reference and for how long.

Who receives an email?

  • If the submission is being made by a user who is not logged in, and the submitter has provided an email address, he/she will receive a confirmation email.
  • If the organization being updated has an email address, and it is different from the submitter's address, the organization will receive an email. In addition, the organization will receive an email to both the "old" and "new" address if the submitter has suggested a change to the organization's email address. This email can be suppressed.
  • The email for updates associated with the Record Owner Agency will receive a copy of the suggested changes. This email address can be changed from the Agency setup page. This email can be suppressed.

After Submitting

If all required information from Before Submitting has been filled out appropriately, the user will receive a notification indicating their changes have been accepted. The message the user receives is customizable by the Super User from Setup > General Setup Options. If the Feedback was on an existing record, the user will be redirected back to the Record Details page.
