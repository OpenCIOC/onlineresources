Community Sets allow you to take a selection of Communities and Volunteer Opportunities and make them available in a particular View. You can use a Community Set for multiple different Views, but each View can only have one Community Set.

Each Community Set must have a unique Set Name. The Set Name is visible on data entry forms if you have more than one Set, and is visible as a list of Sets the record belongs to if you have added the Community Set field to the Record Details page in your View. It is used in the View Setup Form to pick the Community Set used for the View, and on the Update Form and Volunteer Opportunity Community Set Management Tool to assign a record to one or more Sets.

The Area Served is seen on the main menu page ("Looking for Opportunities in...") and on the page Search: Step 1. Many different Community Sets can use the same value for their Area Served.

Some databases will only need to have one Community Set. You need more than one Community Set if:

  • You want to list different geographic areas in different Views (such as on search forms), and/or
  • You want different sets of records in different Views (not including showing/hiding non-public and deleted records, etc)

To modify an existing Set, make your changes and click Update next to the Community Set you have changed. You can only change one Community Set at a time. To add a new Set, enter your values at the bottom of the form and click Add. You will only see the option to Delete a Community Set if it is not being used by a View or Opportunity.
