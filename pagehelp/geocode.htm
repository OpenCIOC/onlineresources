The Update Geocoding on Selected Records page uses your choices about mapping type (Current record setting, Site address, Intersection, or No value (do not map)) as well as the mapping category (aka map pin) setting that you selected on the previous page to update the map location and settings of the selected records.

Some important things to note:

  • You are permitted to geocode a maximum of 15,000 locations per day, per IP Address. This would include any locations entered for proximity searching.
  • The geocoding occurs one record at a time. If you leave the page before the status has been set for the very last record on the page, your geocoding will not be completed.
  • If the status column indicates Updated, then the geocoding information in the record has been set (or reset) with a latitude and longitude received from the mapping provider. If a new mapping category or mapping type was selected, those new values have also been updated in the record.
  • If the status column indicates Unchanged, then the geocoding information has remained completely unaltered in the record (including any mapping type or mapping category changes you have requested). This occurs for one of the following reasons:
    • You do not have permission to update the selected record, or
    • You have selected "Current Setting" and the current mapping type is No value (do not map) or Manual placement, or
    • You have selected a mapping type of Intersection or Site address but there are no valid values in those fields.
  • If the status column indicates that the Address cannot be found, then the mapping provider was unable to located the specified location. This may be because some aspect of the address is incorrect (most often street type, street direction, or postal code), so you should review the record for possible errors. Some new or rural locations cannot be mapped automatically; in these cases, use manual placement.
  • If the status column indicates that you should Contact your system administrator, this can often occur if you attempt to map large numbers or records in quick succession. Try mapping in smaller sets of less than 500 records at a time, and leave a few minutes in between requests.
