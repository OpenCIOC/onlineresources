This page is used to either add a new community to the database, or to edit information for an existing community. For the most part, the fields will be identical whether you are adding a new community or editing an existing one. Some additional information is available when editing an existing community. This will be outlined below.

Remember that this is a shared classification system, used by both the Community Information and Volunteer Opportunities modules. Please be respectful of partner agencies when modifying these values.

Alternative Search Areas vs. Standard Hierarchy Communities

There are two types of communities, both of which can be added to search forms and records:

  • Standard Hierarchy Communities - these provide the basic list of Communities, grouping smaller areas into larger ones. Each of these communities has exactly one parent community.
  • Alternative Search Areas - these provide a way to describe areas that do not fit into the hierarchy, for example because they cross boundaries or group a number of regions together for a special designation such as a health service area.

The data entry form on this page changes depending on which type of Community you are editing.

Adding A New Community

Note: Communities that you add to the database other than through an import from the official repository are considered local or unauthorized. Any communities added locally are available in your application, but will not be available in other CIOC databases. In order to "authorize" your communities, please submit your change requests through the CIOC Communities Repository.

Name

Enter the name of the new community here. Note that if you try to enter a community name that is already in use in the database, you will receive an error when you submit the form, and be prompted to return to the form and try again. You may need to be creative with your community names if you find duplicate values in the database. If a multi-lingual version of the software is being used, you may also enter names in other languages if it differs.

Display

You may enter a different version of the Community name to be used on search forms and under the "Located In" value of search results. This is particularly useful if you have had to modify or lengthen the name in order to get a unique name for the community. If the bilingual version of the software is being used, you may also enter a French version of the display name if it differs from the English version.

Parent Community

Every community entered has a parent community, except for the community "highest" on the hierarchy (Unknown). When you enter a new community name, you must also assign a parent community. This information is necessary for "Located In" and "Areas Served" searches in the database. If you do not know what the parent community should be, you can use the community finder tool which is linked from record update/add forms (in the "Located In" and "Areas Served" fields) to determine this information. Select a value from the "Parent Community" drop-down menu to assign to a newly added community name.

Province, State and/or Country

Optionally, you may specify the Province/State and/or Country by selecting from the drop-down list provided. This is only necessary to disambiguate cases where there are multiple Communities with the same name.

Alternate Names

You may specify one or more Alternate Names per Community; these cannot be the same as any other Community name in the database. Alternate Names are not visible anywhere in the software, but are used when searching for Communities such as in the "Other Community" box on the search page or when doing data entry.

Search Communities

When managing an Alternative Search Area, an extra section is added to the form to allow you to select which Communities from the hierarchy will be searched whenever this Alternative Search Area is used. Adding an Alternative Search Area to a record or search form is equivalent to adding all the Communities chosen in this section.

Editing An Existing Community

The form to edit an existing community provides all the fields listed in "Add New" above, as well as a status section and a search link to review the records associated with the given Community.

Status

The Status section indicates how the Community is being used by Organization/Program or Volunteer Opportunity records, other Communities, or related items such as Bus Routes and Wards.

If the community is not being used, a delete button will be available. If you wish to delete this community and there is no delete button available, you will first need to remove the community from any records it is associated with and then return to this page.

Note: Deleting an Authorized community is not recommended. If you wish to suggest changes to Authorized Communities, please submit your change requests through the CIOC Communities Repository.

Working with languages

In multi-language databases, the software will normally default to the regular working languages of your database (generally, English and French). If additional languages are available for editing, you will see checkboxes at the top of the form; select the additional language(s) you wish to edit by checking the box next to the language, and they will be added to the form.

Important note: If you do not specify a value in every language, the first available language name will be used. It is not necessary to duplicate the name for each language, but neither does it cause a problem to do so.
