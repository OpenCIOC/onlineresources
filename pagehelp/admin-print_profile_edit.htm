Print Profiles: the Basics

Creating a print profile will allow you to generate a list of records formatted nicely for printing, with no extraneous menu information, and with whatever fields you choose. Unlike the search results, where the layout is a horizontal table of fields, the Print Tool outputs records in a custom vertical format; fields each appear on their own line (although there is some capability for combining data from multiple fields into one). Print Profiles are associated with particular Views.

The Print Tool is HTML-based. This means that:

  • HTML and CSS (Cascading Style Sheet) knowledge is very helpful in using the more advanced features of the tool. If you do not know HTML, you will still be able to make some use of the tool. You may benefit from other members' assistance in providing sample style sheets (CSS) and setup options.
  • In most cases, record data will appear as it does on the Record Details page. Any fields that accept HTML input for display on the Record Details page will also accept HTML in the Print Tool, and the same formatting rules apply. Any extra HTML that is inserted into the Record Details pages automatically by the software (such as web site linking) will be suppressed when using the Print Tool.
  • There is limited concept of a "page" in HTML, because the display occurs on a screen which can vary between users. Therefore there is limited ability to add page-specific items like page headers and footers, page numbering, etc. Many Web Browsers offer customizable print setup options which may allow you to add some header/footer information, page numbering, and so on.
  • Your Print Tool generated report should be usable in most major Word Processing programs including Microsoft Word. Newer word processing software will better interpret the HTML and styles than will older versions. You can use word processing software to do things like customize page break locations, add page headers and footers, create columns, and add page numbering.

Editing your Print Profile

You print profile is managed in two parts:

  1. Your general settings for the profile (including style sheets (CSS), headers and footers, and so on).
  2. The field setup for the profile, where you select your fields and specify how they are displayed.

This page deals specifically with your general profile settings; help on setting up your fields can be found in the page help on the field setup page. The link to the manage field data can be found at the bottom of the general settings page; you should always submit your changes to the general profile settings prior to clicking the link to manage fields.

General Profile Settings

Name

Each profile must have a unique name. The name should be descriptive enough that the person who wishes to run the report will know which profile to choose. Names can be a maximum of 100 characters long, but it is strongly recommended that names be kept under 50 characters.

CSS / Style sheet Information

To take full advantage of the Print Tools capabilities, you should specify a Cascading Style sheet (CSS) to use with the profile. Two sample style sheets are provided with software, which you can use "as-is", or copy and change to suit your needs. The location of the samples are:

mydatabase.cioc.ca/styles/printSampleA.css
mydatabase.cioc.ca/styles/printSampleB.css

...where mydatabase.cioc.ca is the URL of your CIOC database.

Within your style sheet, you can create CLASS items to specify the format of various elements in the report. From within the general setup, you can specify the CLASS name to use for the HTML TABLE that will contain your data for all non-heading fields (see field setup page help for more information).

Record Separator

You can add HTML to appear in between records to separate them. An example might be a horizontal rule (line) <HR>, or a line break <BR>. You may also specify whether each record should appear on its own page when printing from a browser...but note that this feature may not be available in all web browsers, and does not currently transfer over when opening your report in Word Processing software like Microsoft Word.

Headers and Footers

You can add custom report headers and footers to the profile. Headers and Footers must be HTML formatted to display properly. Note that headers and footers apply to the entire report: the header and footer are each displayed only once.

Default Message

You can add a customizable message to the report. The message can be edited at the time the report is run.

Message Location

You may specify whether the message is printed just once (at the top of the report) or once before each record.

View Information

Select which View(s) will have access to this print profile. The profile will only be available in the Views you specify, and will be available to any logged-in user with access to that View. You can have as many print profiles per View as you wish.
