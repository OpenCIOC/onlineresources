Last Login

This section shows the last date this login was successfully used (if available) and the IP Address used to access the account.

Locked Account

If this account is locked, a section called Locked Account will be available. It will show the last login attempt made on this account and the IP Address used for the attempt. A user with account management privileges can unlock the account by checking the "Unlock this account" checkbox and submitting the form.

Status

Checking the User is Inactive box will change the account status to inactive and the user will no longer be allowed to log in to the database. If the user is currently logged in, they will be logged out the next time they perform any operation in the database. Because the system does not allow you to delete user logins, this is the method that should be used to block an account that is no longer in use.

Agency

Each user account must be associated with an Agency that has been configured in the database. A Super User can add to or edit the list of Agencies by going to the Agencies setup link in the Main Menu.

User Type

User Types represent the security level under which the user will be able to access any particular module. Selecting a User Type will determine which View the user sees when they log in and what permissions they have to access and modify information in the database. User Types are set individually for each module of the database (Community Information and Volunteer). Depending on your permissions, you may not be able to modify the settings for all the modules in your database.

If you leave the User Type blank for any module, the software will give the user the same permissions as a non-logged in user whenever they are working in that module.

You cannot change a user's existing User Type to one that has greater permissions than your own User Type. For example, if you are not a Super User you would not be permitted to grant a user Super User privileges within the database. However, the user's current User Type will always be available in the list and selected by default, even if they have more permissions than you.

A Super User can modify User Types by going to Setup from the Main Menu and then selecting User Types.

Start Page

The Start Page setting allows you to specify which database module (Community Information or Volunteer) and language that the user will be taken to when they first log in to the database. If you do not use multiple languages or multiple database modules, you may not see this account setting.

User Name

Specify a unique login name for this account. The user will use this along with their password to log in to the database.

Name

First and last name of user, as well as initials. Depending on the database settings, initials may be used in various areas of the software to indicate who performed modifications and updates; therefore, all initials should be unique in the database. If you have more than one users with the same initials, use middle names or add some special designation to distinguish between them, such as an Agency Code.

Email

The user's email will be used to notify them in case of a locked account, and in the future may be used for password resets or other account-related security. In the past this was not a required field, so it is possible for older accounts to exist that have no email address associated with them. In such a case, you will not be able to modify the account without first adding an email.

Saved Search

Specify the number of saved searches this user is allowed to have at any one time; this number does not include searches shared with this user by another user.

Account Updates

You may specify whether the user is allowed to update their own basic account information, or must submit a request for a change to their account. In general, it is useful to allow users to maintain their own basic account details except under special circumstances such as a 3rd party account (e.g. leased login) or a shared account. If you disallow self-updating on any part of the user account, please ensure that the setup for this user's Agency includes an email for sending account update requests.

Single Login

Users with a "Single Login" account cannot log in at multiple locations or with multiple browsers at the same time. If a new login occurs, the former login session will be terminated. This is not usually a recommended setting for database managers or other regular database users, but can be useful when providing a 3rd party with a login that you do not want shared by multiple users, or whenever extra security is desired.

Change Password

Change Password allows the person setting up the account to change a user's password. Note: Please follow the instructions around password security very carefully. Passwords created to allow new users to first log in to the database should be changed as soon as the new user is able to do so.

Submit Updates / Reset Form

When you finish entering/editing the information; click on "Submit Updates" to save the information. Clicking "Reset Form" will return the page to whatever settings were displayed when the page was first loaded.
