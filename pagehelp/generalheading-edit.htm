Introduction to General Headings

What are General Headings?

General Headings allow you to classify records within a Publication. In Publication-based Views, Headings may be used to create the "Quick List" search on the basic search form. Headings are also searchable from the Advanced Search form in any View that has access to that Publication. Records must be first assigned to a Publication before they can be indexed to that Publication's General Headings.

It is possible to create custom display fields using the Headings for any given Publication. Like other classifications such as the Thesaurus or Taxonomy, these Heading display fields will be linked to allow users to perform searches for other records that use the given Heading Terms. It is not necessary for the View to be based on the Publication in order to display the Headings field for that Publication.

When using CIOC's print tools, Headings can be used to generate a topical (subject-based) record index. Some features of General Headings are solely for use in the subject index of Print Tools, including the ability to create an "Unused Term" (that points to one or more preferred Headings to be used instead), or a list of Related Terms ("See Also ...").

Taxonomy Headings vs. Regular Headings

"Regular" or traditional General Headings are manually assigned to records. It is up to the data manager to ensure that the indexing of Headings to records is maintained as records are added or changed. There are tools available to assist data managers in keeping this indexing up-to-date, including the ability to synchronize Heading indexing from one Publication to another.

For those that use the AIRS / INFO LINE Taxonomy in their database, a more automated indexing option is available. Taxonomy Headings include a component describing a Taxonomy Search; any record assigned to the Publication that matches the search criteria will automatically be indexed to the Heading. Indexing is automatically updated every time a record is added or removed from the Publication or has its Taxonomy indexing changed. All records in the Publication are re-evaluated for indexing to the Heading whenever the its Taxonomy search criteria is updated. Note: Taxonomy-based Headings cannot be manually assigned to records; they must match the given Taxonomy search criteria to be indexed.

Each Publication's Headings can be made up of a mix of Taxonomy and regular Headings.

General Heading Data Elements

The only required element for a regular General Heading is the Name in at least one language. The only requirement for a Taxonomy-based General Heading is at least one Taxonomy Term; a name must also be specified if the Heading is not set to use the Taxonomy Term Names.

Status

If you have chosen to edit an existing General Heading, this section will display information about that General Heading. If you are adding a new General Heading, this section will not appear. Inside the Status section is information about whether the General Heading is currently in use by any records. The record count given is for the number of records associated with this General Heading in the entire database (not just the current View).

If the General Heading is being used in records, you will see a "Search" link which will allow you to bring up a results set which shows you the records in your current View that belong to this General Heading. If the General Heading is being used you cannot delete it. You cannot delete regular (non-Taxonomy) Headings that are in use by records. If the Heading is unused or Taxonomy-based, you will have the option of deleting the General Heading using the "Delete" button that will appear at the bottom of the form.

Created / Modified Information

Created and Modified dates / names are set automatically by the software as records are added and updated.

Name

General Heading Names must be unique within each Publication, although you can have the same General Heading Name in multiple Publications. If you are working with a Taxonomy-based Heading, you will have the option of setting the Use Taxonomy Term Name option under some conditions (this is not allowed when you are using multiple "Match Any" Terms, or have a mix of "Must Match" and "Match Any" Terms). If you choose the Use Taxonomy Term Name option, there will be no place on the form to enter a different name; the name will always be generated based on the current name(s) of the selected Taxonomy Codes.

Note: It is important to make this a very brief but clear description as it will impact on the width of the drop down menu that appears for the "Quick List".

Used

This does not apply to Taxonomy-based Headings. Only Used General Headings may be manually added to records. If a General Heading is being used by records, it must have a status of Used (this cannot be changed as long as there are records attached).

Note: Unused Headings are only intended for use in the Subject Index of the print tool; please ensure that you specify at least one Related Heading in order for it to appear in the index. The result will be displayed as "[Heading] See [Related Heading]" in the index.

Taxonomy Search

This area displays any existing criteria for a Taxonomy-based Heading. Click the Add or Remove Terms button to modify your search criteria using a search form that functions similarly to the Advanced Taxonomy Search form. Taxonomy-based Terms must have at least one Taxonomy Term listed in this area. Because this search works so similarly to the Advanced Taxonomy Search, the Page Help for that search may be useful for those not familiar with its functionality.

Non-Public

If a Heading is non-public, it will not be available to search in Views which exclude non-public Publications/Headings from the search form. Note: to fully exclude non-public Headings from a View, you must also use the display version of this Headings field which does not include non-public Headings; the version that does include non-public Headings has a field name ending in "_NP".

Display Order

To create sort order of Headings other than alphabetical, you may group them together by display order number. Headings will sort first by their display order number, and then by name.

Heading Group

Headings Groups allow you to cluster a number of different Headings under a broader category. Headings can be assigned to more than one Group. These Groups can optionally be made searchable through the Quick List setup for Publication-based Views; searches for a Heading Group will match records indexed to any of its Headings.

If you use Heading Groups for this Publication, you may assign this Heading to one Heading Group. This will affect how Headings are sorted, displayed and searched, depending on View setup.

Related General Headings

If you are planning to produce a printed directory using the online print tools, you can specify Related General Headings to include when your Subject Index is produced. Unused or Used General Headings that reference Related General Headings will display SEE or SEE ALSO references respectively.
