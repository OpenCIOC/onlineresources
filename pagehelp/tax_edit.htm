Following are the instructions for each section of the View / Edit Taxonomy Term Form. If you are unfamiliar with any of the Taxonomy's terminology or concepts, use an official reference such as the library on 211taxonomy.org.

Note: throughout the form are links to various related Taxonomy Terms and Concepts; you must submit and save any changes you make on the form before following links to other Taxonomy Terms or Concepts, or your changes will be lost.

Status

This information is read-only and indicates whether the Term is being used by Organization/Program records. If a Term is being used, you will not be allowed to delete it.

Created/Modified Information

Created and Modified dates / names are set automatically by the software as records are added and updated.

Code

All Taxonomy Terms must have a unique Code identifying the Term. The Code is made up of 6 Levels, plus the designation of Authorized or Local. When editing this field, there is one box for each Taxonomy Level, and the Authorized status is set in a different field. The overall format of a Taxonomy Code is: AA-####.####-###.##-L (the -L indicates the term is Local/Non-Authorized)

Note that for the Taxonomy structure to work effectively, each new Term needs a parent Term - this means that for a new Level 4 Term with the Code "AB-1111-2222", there must already be a Term with the Code "AB-1111".

Authorization

Indicates whether a Term is an "official" INFO LINE Taxonomy Term, or specific to the local system. Use of Local Terms is strongly discouraged!

Preferred Term

If using a list of Preferred Terms for activation, indicates whether the current Term is in the Preferred Term list. Note that there is a separate tool for updating the list of Preferred Terms, available from the main Manage Taxonomy page.

Term Name

You must specify a unique name for the Term in at least one language. If you do not provide a name for a Taxonomy Term in a given language, it will not be available when searching or indexing in that language.

Source

Indicate the classification system or catalogue from which the term was sourced (i.e. the AIRS/INFO LINE Taxonomy). The list of sources may be customized (see link on main Taxonomy management page).

Definition

Each Term should have an official Definition indicating the exact meaning of the Term and how it is used. If desired, an Alternate Definition can also be provided (further down on this form) which is also available for keyword searches etc, and which is used instead of the official Definition when displayed anywhere other than this form. If you want to alter the Definition of an Authorized Term, you should always change the Alternate Definition rather than the official one.

Facet

You may optionally select one general category that applies to the Term. The list of Facets may be customized (see link on main Taxonomy management page).

Use References

Use References represent alternate wordings and synonyms for an official Taxonomy Term. Use References must be unique - you cannot create a Use Reference that is already being used as a Name or Alternate Name for an existing Term.

Each Use Reference has an Authorized Status, indicating whether the reference came from the official source or is local. Any new references you add should be marked as Local / Non-Authorized.

See Also References

A See Also Reference indicates a sibling-type relationship between two terms (related Terms). The relationship between related Terms is always bidirectional; If you relate Term A to Term B, then Term B will automatically have a See Also Reference listing for Term A. Likewise, if you remove the relationship between A & B, it also removes the relationship between B & A.

Each See Also Reference has an Authorized Status, indicating whether the reference came from the official source or is local, and an Active status, indicating whether or not you want to use this particular reference. If you do not want to accept an Authorized See Also Reference you should set it to Inactive rather than deleting it. Any new references you add should be marked as Local / Non-Authorized.

Related Concepts

A Term may be associated with any number of general concepts or topics called Related Concepts. The list of Related Concepts may be customized (see link on main Taxonomy management page).

Each Related Concept reference has an Authorized Status, indicating whether the reference came from the official source or is local, and an Active status, indicating whether or not you want to use this particular reference. If you do not want to accept an Authorized Related Concept reference you should set it to Inactive rather than deleting it. Any new references you add should be marked as Local / Non-Authorized.

Comments

You may optionally provide data management notes in any of the available languages. This may be useful to indicate why Use References were deactivated or an Alternative Definition provided, for example.

Alternate Name / Alternate Definition

If you do not want to use/display the official Term Name or Definition, you can provide an Alternate Name or Definition in each available language. Alternate Names and Definitions are included in keyword searches of the Taxonomy, and are used instead of the "official" versions in all areas of the software where Terms are displayed.

Icon URL

Optionally, you can provide a link to an image to display beside the Term's name in the Browse by Service Category Search (Public Search). Icons should be small - it is recommended that they be no larger than 20 x 20 pixels.

Active

An inactive term cannot be added to records and will be excluded from all areas of the system except the Basic Taxonomy Search for Super Users. Links to inactive terms will appear in the current Alert colour.

Note that you cannot change a Term's active status if it is being used by Organization / Program records.

Activations can also be managed using the Taxonomy Activation Tool or the Taxonomy Activation Recommendation Report Tool, both of which are available from the main Manage Taxonomy menu.

Record Deletion

You will only see a Delete button at the bottom of the form if the Term is available for deletion (not used by Organization / Program records). You will receive a confirmation request screen when you delete a Term.
