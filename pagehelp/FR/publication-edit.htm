La page « Ajouter / Modifier une publication » vous permet soit de modifier une publication existante, soit d'ajouter une nouvelle publication à la base de données. Si vous modifiez une publication existante, les paramètres existants seront indiqués dans le formulaire. Si vous ajoutez une nouvelle publication, le formulaire sera vide. Le seul paramètre obligatoire d'une publication est le code unique qui la représentera.

Éléments de données d'une publication

Statut

Si vous avez choisi de modifier une publication existante, cette section donne des informations sur cette publication. Si vous ajoutez une nouvelle publication, cette section n'apparaît pas. La section Statut indique si la publication est actuellement utilisée, soit dans une vue de la base de données, soit par des dossiers d'organismes / programmes. Le décompte des dossiers donné est le nombre de dossiers associés à cette publication dans toute la base de données (pas uniquement la vue actuelle).

Si la publication est utilisée dans des dossiers, vous verrez un lien « Rechercher » qui vous permet de retourner un ensemble de résultats affichant les dossiers dans votre vue actuelle qui appartiennent à cette publication. Si la publication est utilisée, soit dans une vue de la base de données, soit par des dossiers d'organismes / programmes, vous ne pouvez pas la supprimer. Si elle n'est pas utilisée, vous avez le choix de supprimer la publication en utilisant le bouton « Supprimer » en bas du formulaire.

Renseignement de création / modification

Les dates de création et de modification et les noms sont établis automatiquement par le logiciel lorsque des dossiers sont ajoutés et mis à jour.

Code

Le code de publication est la « forme courte » unique et descriptive qui identifie la publication. Seuls les caractères alphanumériques et le tiret « - » sont autorisés dans ce champ et la longueur maximale du code est de 20 caractères.

Notes de publications

Il est recommandé de saisir dans ce champ une description de la finalité de chaque code de publication, afin de rappeler la manière dont on l'utilise. Cela empêche toute confusion ou utilisation erronée et facilitera le nettoyage des codes inutilisés dans le futur.

Nom

Les noms sont parfois utilisés en plus ou à la place des codes dans les menus déroulants des publications, comme dans la « liste rapide ». L'utilisation des noms ou des codes (ou des deux) dépend de la page consultée, ainsi que des paramètres de « noms conviviaux » dans la configuration de la vue.

Remarque : il est important que la description soit très courte mais claire, car cela affectera la largeur du menu déroulant qui apparaît dans la « liste rapide ».

Interne

Le fait de cocher cette case empêche la publication d'apparaître dans les listes déroulantes des publications, dans toutes les vues de la base de données qui ne sont pas autorisées à afficher les dossiers internes. Cependant, cela n'empêchera pas le code de publication de s'afficher dans le champ Publication de tout dossier assigné à cette publication.

Utilisation du champ

Il y a deux types de champs de publication : Description (un bloc de texte jusqu'à 8 000 caractères) et En-têtes généraux (un système de classification personnalisé des dossiers appartenant à cette publication). Les en-têtes généraux peuvent être encore divisés en plusieurs sections. La liste complète des types de champs Publication sont :

  • *_DESC : Description de la publication (affichage, rétroaction)
  • *_HEADINGS : En-têtes généraux publics assignés à ce dossier (affichage, rétroaction)
  • *_HEADINGS_NP : Tous les en-têtes généraux assignés à ce dossier (affichage, rétroaction)
  • *_HEADINGGROUPS : Groupes d'en-têtes associés aux en-têtes généraux publics assignés à ce dossier (affichage)
  • *_HEADINGGROUPS_NP : Groupes d'en-têtes associés à tous les en-tête généraux assignés à ce dossier (affichage)

En cochant les cases Autoriser à l'affichage et sur les formulaires, vous rendrez ces champs utilisables à la fois sur la page de « Détails sur le dossier », la page « Proposer une mise à jour » (c'.-à-d. le formulaire de rétroaction) et, dans le cas des champs HEADINGS_NP, le formulaire de mise à jour ordinaire. Si ces cases ne sont pas cochées, le seul endroit où l'on pourra consulter ou modifier les champs de publication sera la page « Mettre à jour les publications » pour chaque dossier individuel (qui n'est pas accessible au public).

Remarque importante : le fait de sélectionner une publication à afficher ne l'ajoute pas automatiquement à toutes les pages ou formulaires ; un super-utilisateur doit aller sur Configuration > Affichage des champs et nommer les nouveaux champs, puis dans la configuration des vues pour l'ajouter aux vues où on souhaite afficher les champs. Cependant, le fait de décocher la case retirera les champs de toutes les pages ou formulaires utilisant la publication.

En-têtes généraux

Initiation aux en-têtes généraux

Que sont les en-têtes généraux ?

Les en-têtes généraux vous permettent de classifier des dossiers dans une publication. Dans les vues basées sur des publications, on peut utiliser les en-têtes pour créer des recherches par « listes rapides » sur le formulaire de recherche simple. Les en-têtes sont également interrogeables depuis le formulaire de recherche avancée dans n'importe quelle vue ayant accès à cette publication. On doit d'abord assigner les dossiers à une publication avant de pouvoir les indexer aux en-têtes généraux de cette publication.

Il est possible de créer des champs d'affichage personnalisé qui utilisent les en-têtes de n'importe quelle publication donnée. De la même manière que les autres classifications comme le Thésaurus ou la Taxonomie, ces champs d'affichage des en-têtes seront hyperliés pour permettre aux utilisateurs de rechercher les autres dossiers qui utilisent les en-têtes généraux en question. Il n'est pas nécessaire que la vue soit basée sur la publication pour afficher le champ En-têtes de cette publication.

Lorsqu'on utilise les outils d'impression CIOC, on peut utiliser les en-têtes pour générer un index des dossiers par sujet (basé sur les sujets). Certaines fonctionnalités des en-têtes généraux servent uniquement à l'index des sujets des outils d'impression, y compris la capacité à créer des « termes non utilisés » (qui pointent vers un ou plusieurs en-têtes préférés à utiliser à la place) ou à une liste des termes apparentés (« Voir aussi... »).

En-têtes taxonomiques et En-têtes ordinaires

Les en-têtes « ordinaires » ou généraux traditionnels sont assignés aux dossiers manuellement. C'est au gestionnaire de données de s'assurer du maintien de l'indexation des dossiers ajoutés ou modifiés avec les en-têtes. Des outils sont disponibles pour aider les gestionnaires de données à garder leur base de données à jour, y compris la capacité à synchroniser l'indexation avec les en-têtes d'une publication à l'autre.

Pour ceux qui utilisent la Taxonomie AIRS / INFO LINE dans leur base de données, un choix d'indexation automatisée est disponible. Les en-têtes taxonomiques comprennent une composante qui décrit une recherche par la Taxonomie ; tout dossier assigné à la publication qui correspond au critère de recherche sera automatique indexé avec l'en-tête. L'indexation est mise à jour automatiquement chaque fois qu'un dossier est ajouté ou retiré de la publication ou que son indexation avec la Taxonomie change. Tous les dossiers de la publication sont réévalués pour indexation avec l'en-tête, chaque fois que les critères de recherche avec la Taxonomie sont modifiés. Remarque : les en-têtes taxonomiques ne peuvent pas être assignés aux dossiers manuellement ; ils doivent correspondre aux critères de recherche sur la Taxonomie pour être indexés.

Les en-têtes de chaque publication peuvent inclure à la fois des en-têtes ordinaires et des en-têtes taxonomiques.

Groupes d'en-têtes généraux

Les groupes d'en-têtes vous permettent de regrouper plusieurs en-têtes différents sous une catégorie plus large. On peut assigner des en-têtes à plus d'un groupe à la fois. De manière optionnelle, il est possible de rendre ces groupes interrogeables par les paramètres de la liste rapide dans les vues basées sur des publications ; les recherches sur un groupe d'en-tête correspondront aux dossiers indexés à l'un de ces en-têtes. On ne peut pas assigner des dossiers à des groupes d'en-têtes directement. Cliquez sur le bouton « Ajouter un groupe » pour ajouter une nouvelle rangée au tableau et créer une nouveau groupe. On ajoute des en-têtes aux groupes en utilisant l'outil des gestion des en-têtes généraux (voir ci-dessous).

Gérer les en-têtes généraux

La section « Gérer les en-têtes généraux » vous dirigera vers une page pour ajouter, modifier ou retirer des éléments de la liste des en-têtes généraux de cette publication. Remarque : assurez-vous de soumettre d'abord toute modification effectuée sur ce formulaire avant d'aller sur la page Gérer les en-têtes généraux ou vos changements seront perdus.

Modifier un en-tête existant

Pour consulter ou modifier les paramètres d'un en-tête existant, sélectionnez le dans la liste, puis cliquez sur « Voir / Modifier l'en-tête ».

Créer un nouvel en-tête

Des liens sont fournis pour créer un nouvel en-tête ordinaire ou un en-tête taxonomique. Le fait de cliquer sur les liens vous dirigera vers le formulaire vide approprié pour créer le nouvel en-tête.

Ajout rapide d'un en-tête général taxonomique

En utilisant la recherche hiérarchique de la Taxonomie, vous pouvez sélectionner plusieurs termes de la Taxonomie à la fois et créer un nouvel ensemble d'en-têtes généraux basés sur les termes sélectionnés. Vous ne pourrez pas ajouter des termes qui existent déjà dans votre liste d'en-têtes (cela comprend les en-têtes ordinaires ayant le même nom que le terme de la Taxonomie). Cet outil créé un en-tête par terme ; il ne prend pas en compte les termes liés ou la création d'en-têtes basés sur plusieurs termes, mais vous pouvez modifier la liste des termes plus tard si vous avez besoin d'une recherche plus complexe.

Copier les en-têtes généraux

S'il existe une autre publication dans la base de données qui contient tous ou la plupart des en-têtes que vous souhaitez utiliser pour cette publication, vous pouvez utiliser l'outil de copie des en-têtes généraux pour sélectionner une publication et ajouter certains ou tous les en-têtes de la publication actuelle. Bien que l'indexation par en-têtes taxonomiques débute automatiquement lorsque les en-têtes taxonomiques sont copiés, ce n'est pas le cas des en-têtes ordinaires dont l'indexation est manuelle. Pour les en-têtes ordinaires, cette fonctionnalité est souvent utilisée de manière efficace en combinaison avec l'option Synchroniser les en-têtes d'un dossier, qui fait partie de l'outil de mise à jour en nombre« Ajouter / Supprimer un en-tête » disponible par les résultats de recherche.
