Initiation aux noms de domaines dans CIOC

Lorsqu'une nouvelle base de données hébergée par CIOC est créée, on lui donne un nom de domaine de type votreregion.cioc.ca pour accéder à la base de données. Bien que vous puissiez ajouter de nouveaux noms à votre site au fil du temps, ce nom cioc.ca sera toujours important pour vous connecter à votre base de données, car il offre une page de connexion sécurisée. De nouvelles URL peuvent être ajoutées à votre site, sur demande, soit pour être utilisées dans votre site public principal, soit pour être utilisées comme nom mémorisable pour les portails ou vues à usage spécifique que vous développez. Ces nouveaux noms peuvent être des nouveaux noms de type cioc.ca, des sous-domaines basés sur l'URL d'un autre site existant ou un nouveau nom de domaine enregistré pour l'occasion. Si vous souhaitez ajouter un nouveau nom de domaine à votre base de données, veuillez lire l'article How to add a new Domain (URL) to your database (en anglais) sur le site de la communauté CIOC.

Noms de domaines et sites publics / sites à connexion

Les noms de domaines s'avèrent utiles uniquement pour les sites publics ; il n'est donc pas recommandé d'ajouter de nouveaux noms juste pour des utilisateurs devant se connecter. Si vous devez créer un nouveau nom de domaine pour des utilisations nécessitant une connexion, considérez plutôt un nom de type cioc.ca, afin que les utilisateurs puissent utiliser une URL à connexion sécurisée. Soyez conscient que, contrairement au public, la vue sur laquelle l'utilisateur travaille est déterminée par son authentifiant de sécurité, et non pas le nom domaine utilisé pour accéder au site.

Travailler avec votre liste de noms de domaines

Dans la majorité des cas, vous n'aurez pas à modifier les informations sur cette page. Si vous avez un doute au sujet du changement que vous souhaitez effectuer, demandez plutôt au contact officiel de votre base de données de faire une demande de changement directement auprès des administrateurs techniques de CIOC. Veuillez ne modifier aucune valeur de configuration de votre nom de domaine que vous ne comprenez pas.

Ajouter un nouveau nom de domaine

L'ajout d'un nouveau nom de domaine doit toujours être géré par les administrateurs techniques de CIOC, car cela implique la configuration du serveur web. Veuillez lire l'article How to add a new Domain (URL) to your database (en anglais) sur le site de la communauté CIOC pour plus d'informations.

Modifier la langue par défaut

Pour les sites publics offrant des fonctionnalités de recherche dans plusieurs langues, chaque enregistrement de nom de domaine comprend une indication sur quelle langue doit être utilisée si la langue n'est pas précisée explicitement. Par exemple, monsite.cioc.ca?Ln=fr-CA demande explicitement le français si celui-ci est disponible, alors que monsite.cioc.ca utilisera la langue par défaut sélectionnée lors de la configuration. Pour que cette option s'avère utile, la vue doit être configurée de manière à être accessible dans cette langue. Il est recommandé de sélectionner la langue principale de la communauté que vous ciblez.

Modifier la vue

-- Vue par défaut de la base de données --

En sélectionnant cette option, l'utilisateur sera automatiquement dirigé vers la vue actuellement sélectionnée comme vue publique par défaut. Il est fortement recommandé de sélectionner cette option, plutôt que d'aller sélectionner le nom de votre vue publique par défaut dans la liste.

Nom spécifique de la vue

Sachez qu'il existe des vues non-publiques dans cette liste, indiquées par un *. Cela peut s'avérer utile pour vous aider à préparer avant le lancement, mais les utilisateurs du public seront dirigés vers la vue publique par défaut si vous sélectionnez une vue non-publique. En aucun cas vous ne devez choisir une vue non-publique de manière intentionnelle - n'oubliez pas que les utilisateurs connectés obtiendront la vue spécifiée dans la configuration pour leur type d'utilisateur et que les utilisateurs du public ne peuvent pas accéder aux vues privées ; ce choix ne s'avère donc pas utile.

Noms secondaires

Si vous avez plusieurs noms de domaines renvoyant vers la même vue, vous devriez en sélectionner un comme nom préféré et marquer les autres comme « secondaires » (p. ex., si vous avez monpremiernom.com et mondeuxiemenom.com, ou des variations comme www.monsite.com et monsite.com, qui envoient toutes vers la même chose). Cela évite, par exemple, des dédoublements inutiles dans la liste des domaines / vues lors de l'envoi des demandes de mise à jour par courriel, et cela aide également le logiciel à sélectionner un nom préféré lors de la création de liens pour les courriels sortants (p. ex., courriels des profils bénévoles).

Google Maps

Des clés API Google Maps ou les renseignements d'identifiant du client Enterprise ou Enterprise Channel sont requis afin d'utiliser l'outil Google Maps intégré à CIOC. Lorsque vous demandez qu'un nouveau nom de domaine soit ajouté à votre site, vous pouvez demander aux administrateurs techniques de CIOC qu'ils configurent également la clé Google Maps, si vous êtes admissibles (au minimum, le site accessible au public d'un organisme sans but lucratif ou caritatif). Si vous le souhaitez, vous pouvez demander vous-même une clé API pour votre nom de domaine autre que cioc.ca, et le saisir ici. Si vous n'êtes pas admissible à une clé API gratuite et si vous détenez un compte Enterprise auprès de Google, vous pouvez également utiliser cette zone pour saisir vos informations d'identification client Enterprise Google Maps et Channel.
