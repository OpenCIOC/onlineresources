Information générale sur la modification des listes de contrôle

Il est important de lire attentivement les renseignements contenus dans la page « Modifier les valeurs de la liste de contrôle / déroulante » sur laquelle vous travaillez actuellement.

En fonction du type de liste ou de menu déroulant spécifique que vous avez choisi de modifier, la page affichera des valeurs différentes. La majorité des versions de la page « Modifier les valeurs de la liste de contrôle / déroulante » ont un format et des noms de champs similaires ; ce fichier d'aide aborde les variantes principales que vous pourrez observer. Si vous travaillez dans une base de données multimembre, veuillez lire la section sur la manière de travailler avec des valeurs partagées et locales qui vous aidera à comprendre comment gérer vos listes de contrôle efficacement, en coopération avec les autres membres CIOC de votre base de données.

Remarque : en général, les changements apportés aux valeurs sur ces pages mettront également à jour les valeurs dans les dossiers existants ; les exceptions comprennent les types de rue, les types de boîte postale, les titres de service (places disponibles), et les titres honorifiques des contacts. Les changements apportés aux listes qui font exception ne modifieront pas les dossiers existants.

Travailler dans une base de données multimembre : valeurs partagées et locales

Une base de données multimembre est une base de données dans laquelle plusieurs membres CIOC ont leur propre licence distincte, mais partagent certains éléments de configuration communs et ont la capacité de partager des dossiers. Si vous travaillez dans une base de données multimembre, la configuration des valeurs de vos listes de contrôle et déroulantes devient un peu plus compliquée, car vous pourriez partager certaines ou toutes les valeurs avec d'autres membres de la base de données.

Listes de contrôle entièrement partagées

  • Une liste de contrôle entièrement partagée est une liste dont le contenu ne peut être personnalisé pour chaque membre - tous les membres de la base de données doivent utiliser les mêmes valeurs. Certaines de ces valeurs sont pas personnalisables par l'utilisateur pour le moment (valeurs fournies par CIOC), donc tous les utilisateurs utilisent présentement les mêmes valeurs dans leur base de données distincte.
  • Les valeurs modifiables (non fournies par CIOC) ne peuvent être modifiées que par un super-utilisateur global. Les super-utilisateurs locaux peuvent consulter la liste des valeurs d'une liste de contrôle entièrement partagée, mais ne peuvent pas les modifier. Les super-utilisateurs locaux peuvent utiliser le lien fourni pour envoyer un avis aux super-utilisateurs globaux leur demandant de modifier ou de créer une valeur.
  • Ceci inclut : Orientation, Type de rue, Type de boîte postale, Statut des activités, Type d'adresse de facturation, Titres honorifiques, Types de téléphone des contacts, Catégories de cartographie, Domaines d'intérêt général.

Listes de contrôle partiellement partagées

  • Une liste de contrôle partiellement partagée est une liste sur laquelle chaque membre a un certain contrôle des valeurs qu'il souhaite utiliser. Certaines valeurs peuvent être partagées par tous les membres, certaines peuvent être utilisées par seulement quelques membres, et d'autres peuvent être spécifiques à un seul membre.
  • Seul un super-utilisateur global peut créer ou modifier des valeurs partagées accessibles à tous les utilisateurs de la base de données. Un super-utilisateur local peut choisir de ne pas utiliser de valeurs partagées en masquant cette valeur aux autres membres.
  • Les super-utilisateurs locaux peuvent ajouter des valeurs locales qui n'existent pas déjà. Le super-utilisateur local a le contrôle exclusif de cette valeur locale ; elle ne pourra pas être utilisée ou modifiée par d'autres membres. Si un membre veut utiliser une valeur locale créée par un autre membre, un super-utilisateur global peut la convertir en valeur partagée. Les super-utilisateurs locaux peuvent utiliser le lien fourni pour envoyer un avis aux super-utilisateurs globaux leur demandant de modifier ou de créer une valeur.
  • Les valeurs de liste de contrôle non utilisées par le membre actuel continuera d'apparaître dans le contenu des dossiers, les formulaires de mise à jour, etc., si le dossier en question utilise déjà ces valeurs. Ce membre n'aura pas accès aux valeurs non utilisées pour les ajouter aux dossiers qui n'utilisent pas déjà ces valeurs.
  • Ceci inclut : Accessibilité, Accréditation, Secteurs desservis, Lignes de bus, Certification, Distribution, Liste de contrôle supplémentaire A, Liste de contrôle supplémentaire B, Liste de contrôle supplémentaire C, Liste de contrôle supplémentaire D, Liste de contrôle supplémentaire E, Liste de contrôle supplémentaire F, Liste déroulante supplémentaire A, Liste déroulante supplémentaire B, Liste déroulante supplémentaire C, Liste déroulante supplémentaire D, Liste déroulante supplémentaire E, Liste déroulante supplémentaire F, Liste déroulante supplémentaire G, Liste déroulante supplémentaire H, Honoraires, Fin d'année financière, Bailleurs de fonds, Langues, Cartographier le dossier, Type de membre, Mode de paiement, Conditions de paiement, Devise préférée, Qualité du dossier, Type de dossier, Écoles, Niveau de service, Type de garde, Type de programme, Titre du service (places disponibles), Public cible (places disponibles), Type d'unité (places disponibles), Quartier

Modifier des valeurs

Boîtes de saisie : noms, codes et valeurs

Chaque type de liste de contrôle possède un ou plusieurs éléments constituant chaque valeur. Certaines valeurs modifiables ne contiennent qu'une boîte de saisie, en général pour le nom (p. ex., Accessibilité), alors que d'autres valeurs contiennent deux ou plusieurs boîtes de saisie pour chaque valeur modifiée (p. ex., Distribution contient à la fois une boîte pour le code de distribution et une autre pour le nom complet de la distribution). Dans les bases de données multilingues, la majorité des boîtes de saisie possèdent un espace pour saisir les traductions du nom.

À noter que dans les cas où la saisie d'un code ou d'un numéro est offerte, cela reste facultatif et peut même ne pas être utilisé à l'affichage. La saisie d'un code ou d'un numéro est obligatoire pour les listes de contrôle suivantes :

  • Ligne de bus
  • Devise
  • Qualité du dossier
  • Type de dossier
  • Niveau de service
  • Quartier

Travailler avec les langues

Pour la majorité des listes de contrôle, vous pouvez préciser les valeurs de chaque élément dans plusieurs langues. Pour d'autres, vous devez sélectionner une langue unique pour chaque élément de liste.

Pour les listes multilingues, le logiciel utilisera par défaut les langues de travail normales de votre base de données (en général, anglais et français). Si des langues supplémentaires peuvent être modifiées, vous aurez accès à des cases à cocher en haut du formulaire ; sélectionnez les langues supplémentaires que vous voulez modifier en cochant la case à côté de la langue pour l'ajouter au formulaire.

Remarque importante : pour la majorité des listes de contrôle, vous devez préciser un nom dans chaque langue pour que cette valeur apparaisse lorsqu'un dossier est affiché dans cette langue. Les exceptions comprennent :

  • Ligne de bus - utilise le numéro d'itinéraire
  • Langues - utilise le premier nom disponible s'il n'est pas disponible dans la langue de travail
  • Devise préférée - utilise un code
  • Qualité du dossier - utilise un code
  • Type de dossier - utilise un code
  • Écoles - utilise le premier nom disponible s'il n'est pas disponible dans la langue de travail
  • Niveau de service - utilise un code
  • Type d'unité (places disponibles) - utilise le premier nom disponible s'il n'est pas disponible dans la langue de travail
  • Quartier - utilise un code

Lorsque vous devez préciser un nom afin de pouvoir utiliser l'élément de liste de contrôle dans cette langue, la boîte est surlignée en rouge si elle est vide.

Ordre

Pour de nombreux types de listes de contrôle et déroulantes, il existe une colonne « Ordre » dans la page de modification des valeurs. L'ordre d'affichage déterminera l'ordre dans lequel chaque valeur apparaîtra dans la liste sélectionnée, ainsi que dans le dossier. Les numéros d'ordre d'affichage peuvent être dupliqués ; lorsque deux entrées ont le même numéro d'ordre d'affichage, elles sont classées par ordre alphabétique du nom ou du code fourni.

Ajouter, mettre à jour et supprimer

Toutes les valeurs du formulaire de liste de contrôle sont ajoutées, mises à jour ou supprimées en une seule soumission. Il faut utiliser le bouton « Mettre à jour » pour accepter tous les changements. Souvenez-vous : vous devez toujours soumettre tous les changements par le bouton « Mettre à jour », qu'il s'agisse d'un ajout, d'une mise à jour ou d'une suppression.

Ajouter : Pour créer une nouvelle entrée, cliquez sur le bouton « Ajouter un nouvel élément » pour insérer une rangée et y saisir de nouvelles données. Vous pouvez saisir plusieurs rangées par soumission.

Supprimer : Pour supprimer une entrée existante, cochez la case dans la colonne « Supprimer ». S'il n'y a pas de case à cocher, c'est que l'élément est utilisé et ne peut être supprimé pour le moment.

Listes de contrôle Type de rue, Type de boîte postale et Titres honorifiques des contacts

Ces éléments de liste de contrôle fonctionnent de manière différente que les autres types de listes déroulantes ; ils ne sont pas rattachés à des langues et constituent seulement des valeurs suggérées pour les dossiers. Il est possible que votre base de données contiennent des dossiers utilisant des valeurs différentes de celles des listes - même si généralement, il n'est pas possible de sélectionner une nouvelle valeur ne faisant pas partie de la liste - et les changements apportés à la liste n'affectent pas les dossiers existants. De plus, les valeurs de ces listes peuvent être utilisées dans n'importe quelle langue d'un dossier, même si elles sont spécifique à une langue particulière, car ces valeurs ne sont pas automatiquement déterminées par la langue.

Chercher et créer des rapports sur les valeurs des listes de contrôle

Recherche et utilisation

La plupart des listes de contrôle ont une icône de recherche . Le fait de passer la souris au-dessus de l'icône affiche l'usage actuel de cette valeur dans la base de données, y compris l'utilisation dans des dossiers supprimés ou expirés. Le fait de cliquer sur l'icône lancera une recherche sur les dossiers utilisant cette valeur dans la vue actuelle (y compris les dossiers supprimés ou expirés s'ils sont visibles dans la vue). Si vous voyez l'icône de recherche pour certaines valeurs, mais pas pour d'autres, cela signifie que les valeurs sans icône ne sont utilisées dans aucun dossier.

Rapports

La majorité des listes de contrôle comportent un lien vers une version imprimable qui, pour la plupart des listes, inclue toutes les valeurs, ainsi que la date de modification, qui a effectué la modification et le nombre d'utilisations.

Modifier les groupes d'âge

La page Modifier les groupes d'âge est un outil administratif pour créer des classes d'âge avec un nom personnalisé ; on peut les utiliser dans la recherche simple pour identifier les dossiers desservant cette classe d'âge.

Pour permettre à l'utilisateur final de rechercher sur des groupes d'âge, il faut procéder en trois étapes :

  1. Remplir le champ Admissibilité de chaque dossier de la base de données avec les âges minimal et maximal adéquats.
  2. Créer des groupes d'âge avec un nom par la page Modifier les groupes d'âge
  3. Activer l'option de recherche par âge dans la vue où vous voulez qu'ils apparaissent.

Le tableau sur cette page contient les colonnes suivantes :

  • Nom du groupe d'âge : p. ex., Nourrissons, Enfants, Jeunes, Adultes, Aînés
  • Âge minimal : L'âge minimal de ce groupe d'âge. À noter que si cette colonne est vide, cela indique que l'âge minimal est 0. Les décimales sont autorisées.
  • Âge maximal : L'âge maximal de ce groupe d'âge. L'âge précisé ici est inclus dans la classe . De nouveau, si c'est vide, c'est qu'il n'y pas d'âge maximal pour ce groupe d'âge. Les décimales sont autorisées.
  • Garderie : indique si le groupe d'âge doit être utilisé pour la page de recherche spéciale aux garderies.

Remarque importante : la composante d'âge du champ Admissibilité dans chaque dossier de la base de données doit être renseigné correctement avec des données minimales et maximales ; autrement, les résultats des recherches basées sur l'âge seront inexacts.
