Les options de recherche sur cette page sont établies par votre type d'utilisateur ainsi que les paramètres de votre vue actuelle.

Mots de recherche

Le moyen de recherche principal dans le logiciel est la recherche par mots-clés. Quatre champs de recherche par mots-clés potentiels sont disponibles, en fonction de ce qui a été rendu disponible sur votre formulaire de recherche actuel. Ils comprennent :

  • Nom(s) d'organisme - Tous les niveaux hiérarchiques de l'organisme, ainsi que les noms alternatifs et antérieurs.
  • Sujets - Tous les sujets locaux et autorisés, ainsi que les termes « employés pour » associés aux sujets et tous les termes génériques de ces sujets.
  • Catégories de service - Tous les termes de la Taxonomie AIRS / InfoLine indexés aux dossiers, ainsi que les synonymes de ces termes et les niveaux supérieurs, à l'exclusion des niveaux 1.
  • Mots-clés - Toutes les options ci-dessus, ainsi que les régions desservies, les notes sur l'adresse, l'édifice, les noms des personnes contact et des dirigeants, l'admissibilité, l'accessibilité, le transport, les frais, les bailleurs de fonds, les langues, la description, la description supplémentaire, les codes SCIAN, le type de programme, le type de garde, les écoles avoisinantes et tous champs supplémentaires configurés pour être intégrés à la recherche par mots-clés.

Contrairement à la page de recherche simple, qui cherche toujours tous les mots et phrases saisies, la recherche avancée vous permet de préciser si vous cherchez Tous les mots (p. ex., la chaîne de recherche chat chien cheval cherchera chat ET chien ET cheval) ou N'importe quels mots (p. ex., la chaîne de recherche chat chien cheval cherchera chat OU chien OU cheval).

La troisième option, Booléen, n'est pas accessible dans toutes les vues et exige une certaine connaissance de la recherche booléenne ; elle permet d'inclure les mots-clés AND, OR et NOT dans votre recherche pour formuler votre propre chaîne de recherche personnalisée. Le logiciel ne corrigera pas les erreurs de votre recherche ; vous devez donc structurer votre chaîne de recherche correctement pour effectuer ce type de recherche. Tous les mots et phrases doivent être séparées par AND, OR, AND NOT, ou OR NOT. Vous pouvez également utiliser des parenthèses pour regrouper différentes sections de votre recherche. Par exemple : (chats AND chiens) OR (cochons AND vaches) OR "singes rouges" ou croix AND NOT rouge.

Les utilisateurs avancés peuvent également utiliser la fonction SQL Server spéciale FORMSOF à l'intérieur d'une recherche booléenne qui permet de chercher les versions d'un mot. Par exemple, FORMSOF(INFLECTIONAL, garde) retournera les différentes versions du mot « garde » : « garderie », « gardien », et « gardiennage ». De la même manière, FORMSOF(THESAURUS, docteur) ** retournera les autres synonymes du mot « docteur », comme : « médecin ».

** Remarque : l'utilisation du THÉSAURUS dépend de la création d'un fichier de thésaurus pour chaque langue dans SQL Server. Comme CIOC n'a pas encore créé de fichier de thésaurus pour aucune langue sur ses serveurs, cette fonctionnalité ne peut être utilisée efficacement pour le moment.

Information générale sur la correspondance des mots-clés

Le logiciel cherche automatiquement les différentes formes d'un mot dans la langue utilisée. Par exemple, animal ou animaux produit le même résultat ; on peut donc utiliser l'un ou l'autre. Vous pouvez empêcher la recherche sur les formes d'un mot en utilisant les guillemets anglais (voir plus bas).

Remarque importante : si un dossier a un profil confidentiel, les champs marqués comme confidentiels sont toujours exclus de la recherche par mots-clés, même si vous avez l'autorisation d'accéder aux données confidentielles.

Utilisation des astérisques métacaractères

Vous pouvez utiliser un astérisque à la fin de tout mot de recherche, par exemple :

  • enfan* trouvera enfant ; enfants ; enfance ; enfantin
  • aggr* trouvera agression ; agressé ; agresseur
  • enfan* aggr* trouvera n'importe quelle des combinaisons mentionnées ci-dessus

Dans de nombreux cas, il est plus efficace de laisser la fonction intégrée du logiciel rechercher les formes d'un mot, que d'utiliser l'astérisque pour les variantes simples du même mot (enfant et enfants, aide et aider). Cela réduit le nombre de correspondances non désirées, comme hand* retrouvant handball, alors qu'on souhaite seulement retrouver les correspondances à handicap, handicapés et handisport.

Utilisation des guillemets anglais

Vous pouvez chercher des mots entre guillemets anglais... ainsi : "clinique médicale" retournera tous les dossiers qui ont exactement des deux mots de manière adjacente, dans le même ordre et dans le même champ.
"acti* soci*" retournera les dossiers comportant "action sociale" ou "activités socialisantes" or "actionnariat sociétaire" dans un champ unique, de manière adjacente et dans le même ordre.

À noter que l'utilisation des guillemets anglaises désactive la capacité du logiciel à recherche les formes d'un mot (comme mentionné plus haut). Ainsi, une recherche sur "animal" ne trouvera pas "animaux".

Menu latéral des sujets

Si le thésaurus est utilisé, l'option menu latéral des sujets est sélectionnée par défaut. Si sélectionné, le menu latéral des sujets affichera un encadré des entrées correspondantes au thésaurus des sujets, chaque fois que vous effectuez une recherche par Sujets ou Mots-clés.

Communautés

Vous permet d'établir les paramètres relatifs aux résultats basés sur « Situé dans la communauté » ou « Desservant la communauté ». En fonction de la configuration de votre vue actuelle sur la base de données, vous pourriez avoir accès à la boîte de recherche « Autre communauté » pour sélectionner une communauté qui n'apparaît pas dans la liste de contrôle pré-établie. À noter que vous devez saisir les noms de communautés exactement comme ils apparaissent dans la liste des noms dans la base de données ; si vous ne connaissez pas le nom correct de la communauté à utiliser, utilisez le lien du Localisateur de communauté pour rechercher le nom de la communauté appropriée. Si JavaScript est activé, vous devriez voir des suggestions de noms de communauté après avoir saisi quelques caractères.

À noter que les dossiers dont les renseignements Régions desservies ou Situé dans la communauté ne sont pas correctement saisis peuvent causer des problèmes au cours de votre recherche. Les dossiers qui n'ont pas de renseignements Régions desservies n'apparaîtront jamais dans les résultats de recherche portant sur Desservant la communauté . Les dossiers qui n'ont pas de renseignements Situé dans la communauté apparaîtront toujours dans les résultats de recherche portant sur Situé dans la communauté, car il est entendu qu'ils n'ont pas d'emplacement spécifique. Pour assurer de bonnes recherches, assurez-vous toujours que les deux types de renseignements Situé dans la communauté et Régions desservies sont complets et exacts dans tous les dossiers.

Situé près de

Saisissez une adresse, un code postal, une intersection ou un point de repère important. Vous pourriez devoir inclure la ville ou la province pour clarifier l'emplacement à utiliser lorsqu'il existe plus d'un emplacement de ce type dans le monde. Après avoir saisi votre emplacement, cliquez sur l'icône pour cartographier votre emplacement. Par défaut, si vous saisissez un emplacement, les résultats de recherche seront triés du plus proche au plus éloigné (les emplacements non cartographiés seront placés à la fin de la liste). De manière optionnelle, vous pouvez annuler ce tri, préciser si les résultats doivent être situés dans un certain rayon autour de l'emplacement cible, ou préciser si les dossiers non cartographiés doivent être inclus dans la recherche.

Âges

Vous permet de préciser un âge spécifique (dans le champ Admissibilité) pour affiner votre recherche basée sur les âges desservis. Pour rechercher les dossiers correctement, vous devez renseigner les champs Âge minimum et Âge maximum dans vos dossiers. Lorsqu'un dossier ne précise pas d'âge minimal et maximal, il est admis que tous les âges sont desservis.

Pour recherche un âge unique, saisissez l'âge dans la première boîte. Le champ accepte les nombres entiers et également décimaux, par exemple 2.5 ans.

Pour recherche une classe d'âge, saisissez la limite inférieure dans la première boîte et la limite supérieure dans la seconde boîte. Vous pouvez préciser si l'âge doit correspondre à une partie ou exactement à la classe d'âge. Afin d'illustrer la différence entre ces deux options :

Admettons qu'une classe de 4-6 ans est recherchée, et qu'il existe 4 dossiers dans la base de données : un desservant les 4 ans et moins, un desservant les 2-6 ans, un desservant les 5 ans et plus, et un desservant Tous les âges (aucun minimum et maximum précisé). Lors d'une recherche doit correspondre à une partie de la classe d'âge, les 4 dossiers correspondent, car ils chevauchent tous la classe 4-6 ans. Lors d'une recherche doit correspondre à la classe d'âge exactement , seuls les âges 2-6 et Tous les âges correspondent, car ce sont les deux dossiers qui desservent chaque âge entre 4 et 6 (inclusivement).

Disponibilité

Cette recherche retourne seulement les dossiers comportant des renseignements sur la capacité ; si aucun public cible n'est associé à la capacité du service, il est admis que le dossier dessert tous les publics cibles.

Nombre d'employés

L'exactitude de la recherche par Nombre d'employés dépend de la fiabilité des renseignements précisés dans vos dossiers ; cette recherche ne retournera que les dossiers dont les renseignements pertinents au nombre d'employés sont indiqués.

Recherche sur les champs Date « personnalisés »

Vous pouvez sélectionner jusqu'à deux champs de date pour la recherche. Votre recherche peut limiter la date à une classe prédéfinie dans la liste déroulante, par exemple « dans le passé », « dans le futur », « dans les 7 derniers jours », etc., ainsi que « Est nul » (les champs sélectionnés doivent être vides) et « N'est pas nul » (les champs sélectionnés ne doivent pas être vides). Vous pouvez également sélectionner des limites de date supérieure et inférieure en indiquant des dates spécifiques. Ceux sont deux entrées disponibles pour créer une recherche personnalisée sur les dates ; si vous créez une recherche pour les deux entrées, les deux doivent correspondre.

La liste des dates est créée à partir des types de dates disponibles dans le logiciel. Si vous avez renommé vos champs de date différemment des noms normalisés livrés avec le logiciel, ou avez ajouté vos propres champs de date « supplémentaires », vos noms peuvent être différents de la liste ci-dessous :

  • Date de cueillette : date à laquelle l'information a été collectée (établie par l'utilisateur)
  • Date de création : date à laquelle le dossier a été créé (établie par le système)
  • Date de suppression : date à laquelle le dossier a été supprimé ou date de suppression prévue (établie par l'utilisateur)
  • Date d'importation : date à laquelle le dossier a été importé dans la base de données (établie par le système)
  • Date de modification : date à laquelle le contenu principal du dossier a été modifié (établie par le système)
  • Date de modification (Taxonomie) : date à laquelle la liste des « catégories de services » du dossier a été modifiée (établie par le système)
  • Date de mise à jour : date à laquelle le dossier a été entièrement mis à jour (établie par l'utilisateur)
  • Dernière demande - Courriel : date à laquelle le système a envoyé une demande de mise à jour du dossier par courriel pour la dernière fois (établie par le système)
  • Calendrier des mises à jour : date prévue de la prochaine mise à jour complète (établie par l'utilisateur)

En plus des dates énumérées ci-dessus, sachez que l'information de Publication contient sa propre date de modification en fonction du code de publication individuel. Ces dates ne sont pas accessibles dans la recherche personnalisée sur les dates et, pour le moment, on peut seulement les rechercher par une recherche SQL. À noter que dans de nombreux cas, les dates établies par le système sont horodatées, même si ce n'est pas visible par l'utilisateur final ; les recherches ci-dessus prennent en compte l'horodatation si elle existe.

Recherche sur les champs Texte « personnalisés »

Vous pouvez effectuer jusqu'à deux recherches personnalisées sur les champs de texte. La liste des champs disponibles pour la recherche est établie en fonction des champs de texte disponibles sur la page Renseignements sur le dossier de la vue actuelle. Vous pouvez utilisez seulement l'un des deux menus de recherche personnalisée ou les deux à la fois ; si vous utilisez les deux, alors le dossier doit correspondre aux deux recherches en même temps.

Il existe deux menus déroulants qui vous permettent de sélectionner les champs dans la table de la base de données et de retourner des résultats basés sur les champs sélectionnés. Les menus déroulants au centre offrent 4 choix :

  • Contient : les champs sélectionnés des dossiers doivent contenir la chaîne de recherche exacte saisie
  • Ne contient pas : les champs sélectionnés des dossiers doivent ne pas contenir la chaîne de recherche exacte saisie
  • Est nul : les champs sélectionnés doivent être vides
  • Est nul : les champs sélectionnés ne doivent pas être vides

Pour chaque recherche de texte, vous pouvez sélectionner / désélectionner plusieurs champs en maintenant la touche CTRL enfoncée lorsque vous cliquez sur des champs de la liste. Si vous effectuez une recherche où plusieurs champs sont sélectionnés dans une même liste, l'option par défaut est que n'importe quel champ peut correspondre au critère. Si vous souhaitez que tous les champs correspondent au critère, vous devez cocher la case « Il doit y avoir concordance de tous les champs sélectionnés ».

Vous pouvez choisir d'ajouter temporairement les champs recherchés au tableau des résultats de recherche en cochant la case « Inclure le(s) champ(s) sélectionné(s) dans le tableau d'affichage des résultats ». Cela ne modifiera pas vos paramètres Changer les options d'affichage et les champs supplémentaires ne seront affichés que pour le tableau des résultats de cette recherche particulière.

Agence propriétaire de dossiers

La liste des agences disponibles ici est établie en fonction des agences marquées comme détenant des dossiers dans le module actuel (Information communautaire ou Bénévolat), dans la section Agences de la configuration.

Occasions de bénévolat

Si le module Bénévolat est utilisé, on peut rechercher des dossiers qui ont (ou n'ont pas) d'occasions de bénévolat.

  • Offre des occasions de bénévolat - a au moins une occasion de bénévolat associé ; ne fait pas de différence si l'occasion est actuellement accessible (p. ex., expirée, interne, supprimée, etc.).
  • Offre des occasions de bénévolat dans cette Vue - a des occasions qui sont actuellement accessibles dans la vue Bénévolat de la base de données (certaines peuvent être expirées).
  • Offre des occasions de bénévolat en ce moment dans cette Vue - A des occasions dont la date Afficher jusqu'à n'est pas passée et qui sont accessibles dans votre vue Bénévolat actuelle.
  • Offre des occasions de bénévolat actuelles, publiques et non supprimées dans cette Vue - a des occasions disponibles ; les occasions actuelles, publiques et non supprimées signifient généralement qu'elles sont accessibles au public.
  • N'offre aucune occasion de bénévolat - n'a aucune occasion de bénévolat associée (y compris des occasions supprimées, passées ou internes, etc.).

Publications / En-têtes généraux

Le fait d'utiliser les options de Publications vous permet de retourner des dossiers en fonction des codes de publication assignés aux dossiers de la base de données. Vous pouvez utiliser les deux tableaux, « Inclure les publications » et « Exclure les publications », soit séparément, soit en combinaison.

  • Aucun retournera les dossiers qui n'ont pas de code de publication assigné.
  • Avec au moins un code retournera les dossiers qui ont au moins un code de publication assigné.
  • Avec l'un de vous permet de choisir dans la liste une ou plusieurs publications pour retourner des résultats où seulement certaines publications sont assignées.
  • On peut utiliser le tableau « Exclure les publications » pour préciser les publications à ne pas inclure dans les résultats de recherche.

Si certaines des publications accessibles dans votre vue actuelle contiennent des en-têtes généraux, vous pourrez les sélectionner dans la liste déroulante sous la section appelée « En-têtes généraux ». Les paramètres de recherche fonctionnent de la manière mentionnée ci-dessous pour les codes de publication.

Si vous travaillez dans une vue limitée par publication, vous ne verrez que les En-têtes généraux de cette publication particulière, plutôt qu'une liste de codes de publication.

Courriel

Recherche les dossiers en fonction du statut des champs Courriel et Courriel de mise à jour.

  • Tous les dossiers - retourne tous les dossiers, qu'ils aient une adresse courriel ou non.
  • Seulement avec un courriel - retourne les dossiers qui ont soit un Courriel, soit un Courriel de mise à jour.
  • Seulement sans courriel - retourne les dossiers qui n'ont pas de Courriel ou de Courriel de mise à jour.
  • Peut envoyer des demandes de mise à jour - retourne les dossiers qui ont soit un Courriel, soit un Courriel de mise à jour, et dont le champ Bloquer demandes par courriel n'est pas activé.
  • Ne peut pas envoyer de demandes de mise à jour - retourne les dossiers qui n'ont pas de Courriel ou de Courriel de mise à jour ou dont le champ Bloquer demandes par courriel est activé.

Statut public

Langue

Cette recherche ne permet pas de préciser actuellement dans quelle langue le dossier est accessible ; elle identifie uniquement les dossiers qui ont un équivalent dans une autre langue autre que la langue actuelle.

Statut de garderie

Statut supprimé

Si vous avez l'autorisation de consulter les dossiers supprimés, vous pouvez cocher cette case afin de les afficher dans vos résultats de recherche. Cela s'applique uniquement aux dossiers dont la date de suppression est dans le passé. Les dossiers dont la date de suppression est dans le futur (« en attente » d'être supprimés) ne sont pas considérés comme supprimés avant la date. Cette recherche est uniquement visible dans les vues qui incluent les dossiers supprimés.

Listes de contrôle

Vous pouvez recherchez dans plusieurs listes de contrôle. À noter que les listes de contrôle disponibles pour la recherche sont établies en fonction de la configuration de la vue actuelle.
