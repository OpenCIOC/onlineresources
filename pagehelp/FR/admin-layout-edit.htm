Les mises en page de modèles sont, de manière simple, des fragments de HTML qui formeront des sections des pages rendues par le logiciel. Les sections d'une page pouvant être modifiées par une mise en page de modèle sont :

  • En-tête - la section de page venant avant le contenu généré par CIOC. Cela contient généralement les renseignements du menu du haut ou latéral.
  • Pied de page - la section de page venant après le contenu généré par CIOC. Cela contient généralement les menus de pieds de page, les avis de droits d'auteur, etc.
  • Recherche - le corps de la page principale pour chaque module du logiciel. Cela contient généralement ce qui s'appelle habituellement la « page principale » ou la « recherche simple ».

Les mises en page de modèles peuvent comprendre des éléments spéciaux qui sont contrôlés par les paramètres de configuration et de sécurité intégrés à CIOC. Vous pouvez utiliser des balises spéciales dans le HTML de votre mise en page pour déterminer où apparaîtront ces éléments et comment ils seront formatés. Il est fortement recommandé d'utiliser ces balises spéciales, surtout dans les vues en mode connexion. Les balises spéciales et instructions disponibles sont documentées dans une liste plus bas. La liste peut s'agrandir au fur et à mesure que de nouvelles fonctionnalités sont ajoutées au logiciel, mais les balises et instructions existantes disposeront d'un soutien à long terme.

Le meilleur moyen de débuter une mise en page de modèle est de modifier une mise en page de système existante. Les mises en page du système utilisent toutes ou la plupart des balises, instructions et éléments de style disponibles.

À noter que les liens internes du logiciel CIOC programmés en dur (comme les éléments du menu) ne donneront pas accès aux pages que l'utilisateur n'est pas autorisé à consulter. Cependant, on ne devrait jamais employer cette méthode, car les utilisateurs pourraient être perturbés par l'invalidité des liens. De plus, le système pourrait faillir avec le temps ou manquer de nouvelles fonctionnalités.

Certains éléments créés pour la rétrocompatibilité sont obsolètes et ne sont pas expliqués dans ce document d'aide volontairement. Si vous trouvez des éléments non documentés dans une mise en page et avez besoin d'information, veuillez poser votre question sur le babillard CIOC.

Section HTML : aperçu des éléments de mise en page et des instructions

Dans le reste du document, éléments se rapporte à une balise unique sans marque de fin, qui affiche les données à sa place lorsque le modèle est rendu. Les instructions se rapportent à un ensemble de balises (avec marques de début et de fin) qui effectuent certains opérations sur un bloc de texte ou en HTML. Les balises utilisées pour insérer les éléments et les instructions sont incluses entre crochets de cette manière : [TAG]. Les instructions comprennent également la partie spéciale de la balise utilisée pour ouvrir et fermer l'instruction: [IF:TAG], [ENDIF:TAG], [FOR:TAG], [ENDFOR:TAG].

Instructions Si

Les instructions « SI » enserrent une section de HTML qui sera rendue uniquement si une condition particulière est vraie. Par exemple :

[IF:NOT_LOGGED_IN]
<p>Bienvenue, Utilisateur public !</p>
[ENDIF:NOT_LOGGED_IN]

Le système de mise en page fournit certaines instructions pour cette fin spécifiquement, mais tous les éléments de contenu et instructions en boucle peuvent également être utilisés comme instruction SI. Voici un exemple d'instruction en boucle qui n'affiche pas la section du menu s'il n'y a pas d'élément du menu :

[IF:CUSTOM_MENU_ITEMS]
<ul>
	[FOR:CUSTOM_MENU_ITEMS]
	<li class="menu"><a href="[LINK]">[DISPLAY]</a></li>
	[ENDFOR:CUSTOM_MENU_ITEMS]
</ul>
[ENDIF:CUSTOM_MENU_ITEMS]

Voici un exemple de balise d'élément de contenu qui n'affiche pas de section avec le menu déroulant « Changer la vue » s'il n'y a pas de vue vers laquelle basculer :

[IF:CHANGE_VIEW]
<hr>
[CHANGE_VIEW]
[ENDIF:CHANGE_VIEW]

Éléments de contenu

Les éléments de contenu impriment certains renseignements prédéfinis provenant de la base de données, qui peuvent déjà inclure certains formatage HTML. Contrairement aux instructions en boucle qui répètent un nombre de valeurs, chaque balise d'élément de contenu rendra une valeur (ou aucune si aucune n'est définie).

Pour rappel, tous les éléments de contenu peuvent également être utilisés avec une instruction SI comme mentionné plus haut. La condition sera fausse si le contenu est une valeur de type faux, comme NULL (vide) ou 0.

Instructions en boucle

Les instructions en boucle vous permettent de répéter l'instruction sur une liste de valeurs et de rendre le texte ou le HTML pour chaque élément de la liste. Chaque élément aura des éléments de contenu pouvant être rendus sur la page (comme [DISPLAY] ou [LINK]). Ci-dessous est un exemple d'une instruction en boucle créant une liste de tous les éléments du menu des en-têtes (les éléments du menu sont ajoutés par le modèle de conception) :

<ul>
	[FOR:CUSTOM_MENU_ITEMS]
	<li class="menu"><a href="[LINK]">[DISPLAY]</a></li>
	[ENDFOR:CUSTOM_MENU_ITEMS]
</ul>

Il existe une instruction SI spéciale disponible dans une instruction en boucle appelée COUNT. Elle vous permet d'insérer du texte ou HTML spécial seulement à partir du deuxième élément d'une liste. Cela s'avère utile pour créer des divisions entre éléments. Par exemple, l'instruction suivante insérera une balise de nouvelle ligne, uniquement à partir du deuxième élément du menu :

[FOR:CUSTOM_MENU_ITEMS]
[IF:COUNT]<br>[ENDIF:COUNT]<a href="[LINK]">[DISPLAY]</a>
[ENDFOR:CUSTOM_MENU_ITEMS]

Pour rappel, toutes les instructions en boucle peuvent également être utilisés comme instruction SI comme mentionné plus haut. La condition sera fausse s'il n'y a pas d'élément dans la liste.

Balises et instructions d'en-tête

Instructions Si

Pour rappel, toutes les éléments de contenu et instructions en boucle peuvent également être utilisés comme instruction SI comme mentionné plus haut (voir aperçu des balises et instructions)

  • LOGGED_IN - Indique que l'utilisateur actuel est connecté.
  • NOT_LOGGED_IN - Indique que l'utilisateur actuel n'est pas connecté.
  • NOT_PRINT_MODE - Indique que la page n'est pas en mode impression actuellement.
  • PRINT_MODE - Indique que la page est actuellement en mode impression.
  • QUICK_BAR - Indique que la barre rapide doit être affichée.

Éléments de contenu

  • HELP_LINK - L'URL d'un lien vers la page d'aide sur la page actuelle.
  • LOGO - Le logo du site provenant du modèle
  • LOGOLINK - Le lien vers le logo du site provenant du modèle
  • MY_LIST - Le HTML pour afficher les fonctionnalités de Ma liste.
  • NEW_SEARCH - L'URL d'un lien vers une nouvelle recherche dans la langue actuelle.
  • PAGE_HELP_TITLE - Fournit le texte « Aide sur la page » dans la langue actuelle de la page.
  • PAGE_NAME - Le nom de la page actuelle dans la langue actuelle.
  • QUICK_LINKS_TITLE - Le texte « Liens rapides » dans la langue actuelle.
  • SITE_BAR_MENU - SITE_BAR_ITEMS comme liste HTML non ordonnée.
  • SUGGEST_LINK - L'URL d'un lien pour proposer un nouveau dossier dans la langue actuelle.
  • VIEW_NAME - Le nom de la vue actuelle.
  • VIEW_TITLE - Le champ « Titre » de la vue actuelle.

Instructions en boucle

  • SITE_MENU - Les éléments du menu montrés au public et aux utilisateurs connectés sur chaque page (habituellement en haut).
  • SITE_BAR_ITEMS - Les éléments du menu montrés aux utilisateurs connectés sur chaque page (habituellement en haut), y compris leur nom d'utlisateur.

Balises et instructions de recherche

Instructions Si

Pour rappel, toutes les éléments de contenu et instructions en boucle peuvent également être utilisés comme instruction SI comme mentionné plus haut (voir aperçu des balises et instructions)

  • LOGGED_IN - Indique que l'utilisateur actuel est connecté.

Éléments de contenu

  • BASIC_SEARCH - Le HTML complet du formulaire de recherche simple.

  • SEARCH_ALERT - « Message d'alerte de la recherche simple » dans la vue actuelle.
  • SEARCH_RIGHT_CONTENT - « Message de droite de la recherche simple » dans la vue actuelle.
  • SEARCH_LEFT_CONTENT - « Message de gauche de la recherche simple » dans la vue actuelle.

Éléments de contenu CIC uniquement

  • BROWSE_BY_INDUSTRY_URL - L'URL du lien vers la page Parcourir par industrie (peut être utilisée dans un onglet jQuery UI pour aligner le contenu de la page avec la page de recherche). Du contenu est disponible uniquement si le SCIAN est activé.
  • CHILDCARE_SEARCH_TEXT - « Description de la recherche sur les gardes d'enfants » dans la vue actuelle.
  • CHILDCARE_SEARCH_URL - L'URL du lien vers la page Parcourir par industrie (peut être utilisée dans un onglet jQuery UI pour aligner le contenu de la page avec la page de recherche). Du contenu est disponible uniquement si le lien vers le formulaire de recherche avancée sur les ressources de garderie est activé.
  • SERVICE_CATEGORY - Interface utilisateur par défaut pour accéder aux recherches par la Taxonomie. Du contenu est disponible uniquement si l'option « Les éléments du menu et la recherche dans la Taxonomie sont disponibles dans cette vue » est activée dans cette vue.
  • SERVICE_CATEGORY_URL - L'URL du lien vers la page Parcourir par Taxonomie (peut être utilisée dans un onglet jQuery UI pour aligner le contenu de la page avec la page de recherche). Du contenu est disponible uniquement si l'option « Les éléments du menu et la recherche dans la Taxonomie sont disponibles dans cette vue » est activée dans cette vue et que l'utilisateur n'est pas connecté.
  • VOL_LINK - L'URL du lien vers la page de recherche simple du module Bénévolat.
  • VOL_LINK_TEXT - Le texte « menu Bénévolat » dans la langue actuelle.

Éléments de contenu Bénévolat uniquement

  • CIC_LINK - L'URL du lien vers la page de recherche simple du module Information communautaire.
  • CIC_LINK_TEXT - Le texte « Menu CIC » dans la langue actuelle.
  • SEARCH_PROMPT - Le texte « Cherchez-vous des possibilités de bénévolat ? » dans la langue actuelle. Si un ensemble de communautés est configuré pour la vue, alors le texte sera « Cherchez-vous des possibilités de bénévolat dans la communauté ? », où la communauté est la valeur Régions desservies de l'ensemble de communautés configuré dans la vue actuelle.
  • VOL_SEARCH_MENU - Le menu des options de la page de recherche du module Bénévolat, y compris le lien vers la recherche par étape.

Instructions en boucle

  • OTHER_LANGS - Une liste des langues disponibles dans la vue actuelle autres que la langue actuelle. Les éléments incluent ceux pour DISPLAY (le nom de la langue) et LINK (un lien pour basculer vers la page principale dans cette langue).
  • SEARCH_MENU_ITEMS - Articles du menu pour le menu de recherche (normalement à gauche de l'écran), sauf les éléments de menu personnalisables.

Instructions en boucle CIC uniquement

  • BROWSE_ORG_ITEMS - Éléments du menu Parcourir par organisme.
  • BROWSE_SUBJ_ITEMS - Éléments du menu Parcourir par sujet.

Instructions en boucle Bénévolat uniquement

  • BROWSE_BY_OP_ITEMS - Éléments du menu Parcourir par occasion.
  • BROWSE_BY_ORG_ITEMS - Éléments du menu Parcourir par organisme.
