La page Ajouter / Modifier une agence vous permet d'ajouter une nouvelle agence (c'est-à-dire un propriétaire de dossiers) à la base de données ou de modifier une agence existante. La plupart du temps, les informations affichées sont identiques que vous ajoutiez une agence ou modifiez des informations existantes. De l'information supplémentaire est disponible pour modifier une agence existante. Voir plus bas.

Ajouter une nouvelle agence

Membre

Ce champ existe uniquement dans une base de données CIOC à plusieurs membres (une base de données « multimembre »). Ce champ est accessible aux super-utilisateurs globaux (ceux qui contrôlent les éléments de configuration partagés par tous les membres de la base de données). Une fois établie, la membriété d'une agence ne peut être modifiée.

Code d'agence

Ce code est composé de trois lettres majuscules. Ce code sera inséré au début du numéro de chaque dossier appartenant à cette agence. La communauté CIOC tente de maintenir des codes de propriétaire de dossiers uniques parmi ses membres ; il est important de vérifier que vous ne créez pas un nouveau code qui est déjà utilisé par un autre membre.

Informations sur le propriétaire de dossiers

Les informations de cette section sont utilisées pour afficher les coordonnées sur les formulaires de courriel ou de télécopie aux personnes qui envoient des rétroactions sur le dossier ; l'adresse courriel est utilisée par le système pour générer plusieurs types de courriels, y compris les courriels de rétroaction et les demandes de mise à jour. Il faut préciser ces informations dans chaque section de la base de données où le code de propriétaire est utilisé (module Information Communautaire ou Bénévolat).

Courriel de mise à jour :
L'adresse courriel qui sera attachée à toutes les demandes de mise à jour par courriel sortantes pour les dossiers de cette agence, ainsi qu'à d'autres courriels générés automatiquement (p. ex., envois de rétroaction). Si cette adresse courriel n'est pas renseignée, le courriel du module par défaut (comme précisé dans les Options générales de configuration) sera utilisé à la place. Il est fortement recommandé de fournir une adresse courriel pour chaque agence.
Téléphone de mise à jour :
Le numéro de téléphone qui sera attaché aux demandes de mise à jour, aux formulaires de courriel ou de télécopie et aux formulaires de rétroaction pour les dossiers de l'agence.
Téléphone du service de renseignements :
Le numéro que le public doit appeler pour des demandes générales (pas utilisé actuellement dans le logiciel).
N<sup>o</sup> de dossier de l'agence :
OBLIGATOIRE EN TOUT TEMPS ! Chaque agence doit être associée à un dossier de la base de données pouvant fournir un nom et des coordonnées. Le N<sup>o</sup> de dossier correspond au numéro du dossier simplifié (propriétaire du dossier + 4 chiffres). Utilisez le lien « Localisateur d'organisme » pour identifier le n<sup>o</sup> du dossier de l'agence. Saisissez le nom de l'organisme que vous cherchez dans la fenêtre contextuelle du « Localisateur d'organisme », puis copiez le numéro de dossier approprié depuis l'ensemble des résultats retournés. Le dossier sélectionné peut être un dossier interne.

Appliquer les champs obligatoires

Si sélectionné, un utilisateur sera forcé à remplir tous les champs marqués comme requis sur le formulaire de mise à jour d'organisme / programme, afin que la mise à jour soit acceptée.

Mises à jour des comptes utilisateurs

Sélectionnez le comportement par défaut des autorisations de mise à jour personnelle sur les nouveaux comptes de cette agence. S'il existe dans la base de données des comptes pour cette agence où l'utilisateur n'est pas autorisé à mettre à jour ses propres renseignements personnels ou son mot de passe, vérifiez que le champ Courriel est bien rempli afin que l'utilisateur puisse demander des changements à son compte.

Modifier une agence existante

Le formulaire pour modifier un code d'agence existant contient tous les champs listés plus haut dans « Ajout d'une nouvelle agence », ainsi qu'une section Statut et un lien de recherche pour consulter les dossiers associés à l'agence en question. À noter que la recherche ne retournera que les dossiers visibles dans la vuesur laquelle vous travaillez actuellement.

Statut

La section Statut indique le niveau d'utilisation du code d'agence dans la base de données, par des utilisateurs et des dossiers.

Si l'agence n'est propriétaire d'aucun dossier, un bouton Supprimer sera disponible. Si vous voulez supprimer cette agence, mais que le bouton de suppression n'est pas disponible, vous devrez d'abord retirer le code d'agence (propriétaire de dossiers) de tous les dossiers ou utilisateurs auxquels il est associé, puis revenir sur cette page.
