Dernière connexion

Cette section montre la dernière date à laquelle ce compte à été utilisé avec succès (le cas échéant) et l'adresse IP utilisée pour accéder au compte.

Compte verrouillé

Si ce compte est verrouillé, une section nommée Compte verrouillé sera accessible. Elle montrera la dernière tentative de connexion effectuée sur ce compte et l'adresse IP utilisée lors de la tentative. Un utilisateur ayant des privilèges de gestion de compte peut déverrouiller le compte en cochant la case « Déverrouiller ce compte » et en soumettant le formulaire.

Statut

Le fait de cocher la case L'utilisateur est inactif rendra le statut du compte inactif et l'utilisateur ne sera plus autorisé à se connecter à la base de données. Si l'utilisateur est connecté en même temps, il sera déconnecté la prochaine fois qu'il effectue une action dans la base de données. Comme le système ne vous permet pas de supprimer des comptes d'utilisateurs, il faut utiliser cette méthode pour bloquer un compte qui n'est plus utilisé.

Agence

Chaque compte d'utilisateur doit être associé à une Agence ayant été configurée dans la base de données. Un super-utilisateur peut ajouter ou modifier la liste de agences en allant sur le lien Agences dans le menu principal de la configuration.

Type d'utilisateur

Les Types d'utilisateurs représentent le niveau de sécurité avec lequel l'utilisateur peut accéder à un module particulier. La sélection d'un type d'utilisateur établira quelle vue l'utilisateur utilisera lors de la connexion et quelles autorisations il aura pour consulter et modifier des données. Les types d'utilisateurs sont établis individuellement pour chaque module de la base de données (Information communautaire et Bénévolat). En fonction de vos autorisations, il est possible que vous nous puissiez pas modifier les paramètres de tous les modules de la base de données.

Si vous ne renseignez pas le type d'utilisateur pour un module, le logiciel donnera à l'utilisateur les mêmes autorisations qu'un utilisateur non-connecté chaque fois qu'il travaille dans ce module.

Vous ne pouvez pas modifier le type d'utilisateur d'un utilisateur existant pour lui donner plus d'autorisations que votre propre type d'utilisateur. Par exemple, si vous n'êtes pas un super-utilisateur, vous n'êtes pas autorisé à donner des privilèges de super-utilisateur dans la base de données. Cependant, le type d'utilisateur de l'utilisateur actuel sera toujours accessible dans la liste et sélectionné par défaut, même s'ils ont plus d'autorisations que vous.

Un super-utilisateur peut modifier les types d'utilisateurs en allant dans Installation depuis le menu principal, puis en sélectionnant Types d'utilisateurs.

Page de démarrage

La configuration de la Page de démarrage vous permet de préciser vers quel module de la base de données (Information communautaire ou Bénévolat) sera dirigé l'utilisateur, et dans quelle langue, lorsqu'il se connectera à la base de données pour la première fois. Si vous n'utilisez pas plusieurs langues ou plusieurs modules de base de données, vous ne verrez peut-être pas ce paramètre.

Nom d'utilisateur

Donnez un nom d'utilisateur unique à ce compte. L'utilisateur l'utilisera avec son mot de passe pour se connecter à la base de données.

Nom

Prénom et nom de l'utilisateur, ainsi que les initiales. En fonction de la configuration de la base de données, les initiales peuvent être utilisées dans différentes sections du logiciel, pour indiquer qui a effectué des modifications et mises à jour ; ainsi, toutes les initiales doivent être uniques dans la base de données. Si vous avez plus d'un utilisateur ayant les mêmes initiales, utilisez le deuxième prénom ou ajouter une désignation spéciale pour les distinguer, comme le code d'agence.

Courriel

Le courriel de l'utilisateur est utilisé pour l'avertir en cas de compte verrouillé et, dans le futur, pourra être utilisé pour réinitialiser le mot de passe ou dans d'autres cas relatifs à la sécurité du compte. Ce champ n'était pas obligatoire dans le passé ; il donc est possible que des comptes existants plus anciens ne soient pas associés à une adresse courriel. Dans ce cas, vous ne pourrez pas modifier le compte avant d'avoir ajouté une adresse courriel.

Recherche sauvegardée

Précisez le nombre de recherches sauvegardées que l'utilisateur peut conserver en même temps ; ce nombre ne comprend pas les recherches partagées avec cet utilisateur par d'autres utilisateurs.

Mises à jour des comptes

Vous pouvez préciser si l'utilisateur est autorisé à mettre à jour les informations de base de son propre compte ou s'il doit soumettre une demande de modification à son compte. En général, il s'avère utile de laisser les utilisateurs maintenir les détails de base de leur propre compte, sauf dans des circonstances spéciales, comme le compte d'un tiers (p. ex., un compte prêté) ou un compte partagé. Si vous interdisez qu'un utilisateur mette à jour son propre compte, veuillez vous assurer que les paramètres pour l'agence de cet utilisateur comprend un courriel pour l'envoi de demandes de mise à jour du compte.

Compte unique

Les utilisateurs ayant un compte de type « Compte unique » ne peuvent pas se connecter depuis plusieurs endroits ou avec plusieurs navigateurs en même temps. Si une nouvelle connexion survient, la session connectée précédente sera interrompue. Habituellement, ce paramètre n'est pas recommandé pour les gestionnaires de base de données ou autres utilisateurs réguliers de la base de données, mais il peut s'avérer utile lorsqu'on fournit un compte à un tiers qui ne doit pas être partagé avec d'autres utilisateurs, ou dès que des mesures de sécurité supplémentaires sont nécessaires.

Changer le mot de passe

Modifier le mot de passe permet à la personne paramétrant le compte de changer le mot de passe d'un utilisateur. Remarque : veuillez suivre attentivement les instructions relatives à la sécurité du mot de passe. Les mots de passe créés pour autoriser de nouveaux utilisateurs à se connecter à la base de données pour la première fois devraient être changés dès que l'utilisateur en a la possibilité.

Soumettre les mises à jour / Effacer le formulaire

Quand vous avez terminé la saisie / modification des informations, cliquez sur « Soumettre » pour sauvegarder l'information. Le fait de cliquer sur « Effacer » retournera la page vers les paramètres initiaux qui étaient affichés quand la page a été chargée.
