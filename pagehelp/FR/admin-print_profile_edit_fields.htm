Souhaitez-vous obtenir de l'information générale sur les profils d'impression ? Consultez le page d'aide principale sur la modification des profils d'impression.

Ajouter un nouveau champ

En bas de la page se trouve un menu déroulant dans lequel vous pouvez sélectionner parmi tous les champs de rapport disponibles. Vous devez sélectionner un Type de champ pour le champ avant de l'ajouter ; vous ne pourrez pas modifier ce type plus tard. Remarque : il est possible d'ajouter deux fois le même champ au rapport.

Modifier un champ existant

Une fois qu'un champ a été ajouté, vous pouvez configurer le style et le formatage du champ, ainsi que l'ordre dans lequel il apparaît. Vous pouvez modifier ou supprimer le champ, mais vous ne pouvez pas modifier son Type de champ. Si vous devez modifier un champ en un type différent, vous devrez le supprimer puis l'ajouter de nouveau. Des renseignements sur la configuration des champs est disponible plus bas dans la section Configuration des champs.

Ajouter des numéros d'ordre

Vous pouvez ajouter des numéros d'ordre aux champs à imprimer (en partant de 1) en insérant les symboles ## dans les choix de préfixe ou de suffixe de tout champ. Par exemple, le fait d'ajouter "##. " (notez l'espace) dans le préfixe du champ ORG_LEVEL_1 produira cet affichage : "123. Mon nom d'organisme 1". "(##) " produira "(123) Mon nom d'organisme 1". En utilisant des balises HTML et des feuilles de style dans les préfixes et suffixes, il est possibles de créer des mises en pages pour les numéros d'ordre encore plus inventives.

Types de champs

Heading

Un type de champ Titre créé le champ comme en-tête HTML (H1-H6). Un champ titre n'apparaît pas dans un tableau et n'a pas d'étiquette de champ.

Normal

Un type de champ Normal créé le champ en tant que rangée dans un tableau à deux colonnes. La première colonne du tableau contiendra l'étiquette du champ (si vous en précisez une), et la deuxième colonne contiendra le contenu du champ.

De grande largeur

Un type de champ De grande largeur créé le champ en tant que rangée dans un tableau où le contenu chevauche les deux colonnes. Ce type de champ s'avère surtout utile pour les longs champs de type mémo, comme les descriptions. Si vous précisez une étiquette de champ, celle-ci sera imprimée au-dessus du contenu du champ, plutôt qu'à côté.

Suite

Un champ de type Suite doit être utilisé après un champ d'un autre type. Un champ Suite vous permet de regrouper des données de plusieurs champs.

Configuration des champs

Niveau du titre

S'applique aux champs de type Titre uniquement. Accepte les valeurs de 1 à 6 : l'outil établira la valeur par défaut à un titre de niveau 1. Remarque : à moins de le spécifier autrement dans votre feuille de style, un grand numéro de titre indique un titre plus petit.

Étiquette

Le nom convivial du champ. Les étiquettes de champs sont toujours facultatives. Vous ne pouvez pas ajouter d'étiquette à un champ de type Suite ou Titre. Pour empêcher qu'un nom de champ ne se divise sur une nouvelle ligne après un espace, utilisez un espace insécable HTML ( )

Style d'étiquette

Si vous utilisez une feuille de style, la classe CSS qui s'applique au champ.

Séparateur

S'applique uniquement au type de champ Suite. Lorsque vous créez un champ Suite, vous devez préciser un séparateur à afficher entre les contenus des champs de données précédents et le contenu du champ suite (cela peut être un espace, un point-virgule, une virgule, un astérisque, etc.). Les données du séparateur devraient être encodées en HTML si nécessaire. Un séparateur sera imprimé uniquement s'il existe des données à la fois dans le champ de suite et dans le champ auquel il fait suite (remarque : s'il existe plusieurs champs à la suite, le séparateur sera imprimé dès qu'un des champs précédents comporte des données, même si le champ placé immédiatement avant le champ de suite est vide).

Classe CSS de contenu

Si vous utilisez une feuille de style, la classe CSS qui s'applique au contenu du champ.

Contenu si vide

Vous pouvez préciser des données par défaut à imprimer dans les cas où le champ est vide. Si vous ne préciser aucune valeur, le champ vide sera omis lors de la production du rapport. Les données par défaut devraient être encodées en HTML si nécessaire. Si vous souhaitez forcer l'impression d'un champ vide, saisissez un espace insécable en HTML ( ) dans le champ Le contenu si le champ est vide.

En plus des données par défaut, vous pouvez également demander à l'outil d'utiliser les données d'un autre champ si le champ est vide. Cela peut s'avérer particulièrement utile dans les cas des descriptions supplémentaires, de publication ou de catégorie, où vous pouvez utiliser la description principale à la place. Pour insérer les données d'un autre champ, saisissez le nom du champ avec le préfixe @@, p. ex., @@DESCRIPTION ou @@ALT_ORG. Vous devez utiliser le nom normalisé et non-modifiable du champ (le nom normalisé est habituellement affiché en majuscules).

Préfixe et suffixe

Les informations de préfixe et suffixe vous permettent d'ajouter des informations avant et après les données du champ. Les données de préfixe et suffixe sont imprimées uniquement si le champ n'est pas vide. Les données de préfixe et suffixe devraient être encodées en HTML si nécessaire. Remarque : vous pouvez ajouter le numéro de publication/catégorie (le numéro d'ordre du dossier dans le document imprimé) à n'importe quel préfixe ou suffixe, en saisissant ## partout dans le champ.

Ordre d'affichage

L'ordre peut être n'importe quel chiffre entre 0 et 255. Les champs sont ordonnés par ordre d'affichage, puis par leur nom de champ non-modifiable. Le même numéro d'ordre d'affichage peut être utilisé plusieurs fois. Il est fortement recommandé d'espacer les numéros généreusement, pour vous permettre d'insérer de nouveaux champs plus tard.
