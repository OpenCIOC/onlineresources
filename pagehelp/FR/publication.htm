Cette page vous permet de créer un nouveau code pour une publication (un regroupement de dossiers) ou de modifier un code de publication existant. Cette page n'est pas utilisée pour modifier les renseignements de dossiers inclus dans une publication - seulement les informations sur la publication elle-même.

Une publication s'avère utile pour :

  • marquer des ensembles de dossiers que vous devez souvent imprimer en groupe
  • créer un ensemble de dossiers connexes à utiliser dans le menu déroulant « Liste rapide » sur la page de recherche simple
  • créer une vue de la base de données qui ne contient qu'une sélection de vos dossiers (appelé également « vue par publication »
  • créer une classification personnalisée s'appliquant uniquement à la sélection de dossiers (effectué par le système d'en-têtes généraux qui font partie des publications)
  • créer un champ de texte personnalisé s'appliquant uniquement à certains dossiers (effectué par l'utilisation d'un champ Description de publication)

Pour les regroupements de dossiers très simples (exportation de données, partage, utilisation interne, etc.), les codes de distribution peuvent représenter un choix plus adaptés, car ils sont plus simples à gérer.

Listes des publications

Les publications sur cette page sont réparties en trois sections :

  • Liste principale des publications détenues/gérées par vous (votre membriété). Dans une base de données à membre unique, c'est la seule liste disponible.
  • Publications partagées. Dans une base de données multimembre, il s'agit des publications qui peuvent être utilisées par tous les membres de la base de données. Seuls les super-utilisateurs globaux peuvent créer et gérer les publications partagées, mais n'importe quel membre peut ajouter ces publications aux dossiers. Si vous ne souhaitez pas utiliser de publication partagée, vous pouvez la « Masquer » en cochant la case à côté de la publication appropriée et en cliquant sur le bouton « Mettre à jour » en bas du formulaire.
  • Publications détenues par d'autres membres. Dans une base de données multimembre, un lien est fourni pour consulter la liste de codes utilisés exclusivement par les autres membres CIOC d'une même base de données. Les super-utilisateurs globaux peuvent les convertir en publications partagées, ce qui les rendra accessibles aux autres membres.

Les colonnes dans le tableau listant les publications incluent certains ou tous les éléments suivants :

  • Code - Le code alphanumérique unique identifiant cette publication.
  • Nom - Le nom « convivial » associé à cette publication. Si rien n'apparaît dans cette colonne à côté d'une publication, cela signifie qu'aucun nom convivial n'a été créé pour ce code.
  • Interne - Un astérisque apparaîtra dans cette colonne si la publication est marquée comme interne.
  • En-têtes - Si des en-têtes sont associés à cette publication, un lien est disponible pour consulter la liste des en-têtes. En mode d'impression, le nombre d'en-têtes généraux associés à la publication est imprimé à la place du lien.
  • Dossiers (locaux) - Le nombre de vos dossiers associés à cette publication.
  • Autres dossiers - Dans une base de données multimembre, le nombre de dossiers appartenant aux autres membres CIOC dans votre base de données qui sont associés à cette publication.
  • Vue (locales) - Le nombre de vos vues utilisant cette publication pour limiter les dossiers accessibles (une « vue par publication »).
  • Autres vues - Dans une base de données multimembre, le nombre de vues appartenant aux autres membres CIOC dans votre base de données. Seuls les super-utilisateurs globaux peuvent le voir.
  • Liste rapide (locale) - Le nombre de vues dans lesquelles cette publication fait partie de la liste rapide sur la page de recherche principale. Visible uniquement en consultant la version imprimable de la liste des publications.
  • Autre liste rapide - Dans une base de données multimembre, le nombre de vues appartenant à d'autres membres CIOC dans votre base de données, dans lesquelles cette publication fait partie de la liste rapide sur la page de recherche principale. Seuls les super-utilisateurs globaux peuvent le voir, uniquement en consultant la version imprimable de la liste des publications.

Modifier une publication existante

Pour consulter ou modifier les paramètres d'une publication existante, cliquez sur le lien « Modifier » à côté de la publication.

Dans une base de données multimembre, vous pourriez ne pas avoir le droit de modifier toutes les publications. Les restrictions de sécurité suivantes sont en vigueur :

  • Seuls les super-utilisateurs globaux peuvent modifier les publications partagées utilisables par n'importe quel membre de la base de données.
  • Seuls les super-utilisateurs globaux peuvent convertir une publication appartenant exclusivement à un membre en publication partagée.
  • Les super-utilisateurs globaux ne peuvent pas modifier les publications appartenant exclusivement à un membre sans les convertir auparavant en publications partagées.

Créer une nouvelle publication

Cliquez sur le lien « Ajouter une nouvelle publication » en haut de la page pour créer un nouveau code de publication.

Toute nouvelle publication appartient exclusivement au membre qui l'a créée. Dans une base de données multimembre, un super-utilisateur global pourra plus tard la convertir en publication partagée utilisable par d'autres membres.

Imprimer un rapport de vos publications

Le fait de cliquer sur le lien « Version imprimable (nouvelle fenêtre) » ouvrira une liste imprimable de tous les codes de publication dans la base de données.

La page « Liste des publications » fournit une liste imprimable de toutes les publications actuellement accessibles dans la base de données par votre membriété CIOC. Dans une base de données multimembre, ce rapport n'inclut pas les publications appartenant exclusivement à un autre membre. Il inclut certains renseignements non disponibles sur le tableau principal, comme mentionné dans la section précédente qui décrit les éléments du tableau listant les publications.

Cette liste est affichée en « mode d'impression » - si vous disposez d'un modèle d'impression configuré dans votre vue actuelle, ce modèle sera utilisé pour afficher les dossiers.
